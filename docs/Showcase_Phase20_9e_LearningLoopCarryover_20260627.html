Phase20.9e 学习闭环回灌

这一步证明学习闭环不只是面板数字, 它已经能通过 AP 行动竞争影响后继 tick。不会时更想请教, 被教时先整合, 学过后更愿意自己试。

不写死答案 不新增课程脚本 不新增数据库表 只调制行动竞争

在底座中查看

打开工作台:

http://127.0.0.1:8776/phase20_7

发送新问题, 再教学, 再重复同一个问题。观察“学习闭环”和“本 tick 为什么这样做”两块区域。

信息怎样流动

真实 tick外部输入、B/C/C*、行动竞争进入 RuntimeTickEvent。
闭环指标计算反馈、教师退场、冷重测、脚手架四种倾向。
carryover最近 tick 的倾向成为后继 tick 的轻量压力。
行动竞争只改变 drive, 不直接写答案。
后继动作请教、写入、提交、闲时思考自然更容易或更不容易发生。

核心公式: drive_after = clamp01(drive_before + delta(action_type))

你可以怎样验收

1. 发送一个 AP 没学过的问题
   看学习闭环是否偏向“回到脚手架”

2. 在教学纠正里输入一个答案并发送
   看学习闭环是否偏向“先听反馈”

3. 再次发送同一个问题
   看学习闭环是否偏向“尝试自己来”

4. 点开 tick 回放
   查看行动竞争里 request_teacher 或 write_cell 的 drive 是否被 carryover 轻微抬高

边界

这一步可以证明学习闭环已经影响后继 tick 行为。但仍不能声明完整六阶段 runtime、L1/L2/L3 在线嵌入、完整范式自学习、数学列竖式、object-centric 视觉想象或 Phase21 视觉教学泛化闭环已经完成。
