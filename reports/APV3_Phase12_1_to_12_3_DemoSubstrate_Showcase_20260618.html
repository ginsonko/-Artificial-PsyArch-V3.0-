APV3 Phase 12：把机制推进到“可以被看见、可以被试玩”的展示底座

Phase 12 不抢跑 Phase13 课程内容，也不提前做真实硬件。它把 Web 工作台需要的审计 payload、demo profile 和四场景 readiness 先搭好，让后续开源展示不是只靠口头描述，而是可以真实看见内部状态。

验收通过：Phase 11/12 targeted 24 passed，全量回归 409 passed，红线 PASS

三个底座能力

12.1 审计视图把 chat trace、runtime trace、feelings、ledger、tick replay 整理成 phase12_demo，前端可以直接渲染。
12.2 Demo Profile定义四个展示场景，并保留 quiet_girl 风格口味：沉默寡言、惜字如金、可爱但不夸张。
12.3 Readiness Gate用 capability tags 判断场景是否足够可展示，缺能力就列出 missing tags，保持诚实产品化。

四个场景先排好队

文本对话chat / reward / memory / audit，先让用户能玩、能教、能看到学习。
桌宠多模态vision / audio / chat / audit，承接你想要的拟人陪伴方向。
Agent 协作goal / deliberative / trust / audit，让 AP 的心智机制服务工作流。
具身预演vision / action / joint attention / audit，为后续硬件留接口。
Phase 12 的目的不是把 demo 包装得很会演，而是确保它能把真实 AP 状态展示出来。后续 Phase13 课程喂进去以后，用户看到的是“它真的在学”的过程。
