设计闭环

Bn/Cn or work_memory failure
  -> teacher_request SA
  -> external teacher reply_tokens
  -> bridge teaching iteration
  -> AP-native evidence
  -> same cue recall succeeds
  -> later same cue no new teacher_request

审查完善

审查点结论
端到端真实性先真实 recall 失败，再产生 teacher_request，不从已学会状态起跑。
外部教师边界外部教师只提供 reply_tokens；学生侧不读 LLM 策略。
工作记忆时间窗恢复验证放在压力仍有效的短时窗口内，不放宽 runtime。
持续失败达到 max_teaching_iteration_depth 后停止，不无限教学。

最终汇总

可以宣称：主动求教端到端最小闭环已经通过。失败会求教，教师 evidence 会回灌，学会后同 cue 会召回成功并停止重复求教。

仍不能宣称：AP 能自己生成教师答案、完整主动课程规划完成、完整自由中文开放对话底座完成、任意跨模态泛化或 Fresh300 完成。

下一步建议 Phase6.9：做 teacher_request 与持久化恢复 parity，验证 failure/request/teacher evidence/learned paradigm/work_memory item 经 SQLite save/warm-load 后仍能召回且不重复求教。
