## §1 用户洞察 (我的理解)

范式 = 一套"什么时候做什么行动"的内生行动逻辑序列. 关键:
1. **条件是内生感受** (低把握/后继偏置弱/违和/正确感低), 不是外部具体信息 (不是"看到3就写3")
2. **可填入不同对象** = 举一反三 (同一竖式范式可填5+8/任何数字; 同一造句范式可填不同词)
3. **自发发现** = 六阶段学习协议 (模仿→预测→聚合→范式绑定→组织→精修)
4. **人类怎么学范式AP就怎么学** — 观察/模仿/按六步顺序, 不是硬编

## §4 范式发现的AP落地路径 (数学+逻辑推理)

### 理论可行性推理
AP能否自己发现范式? 能. 路径:
1. 教师示范竖式过程 → AP通过echo_imitation模仿 (逐格写)
2. 多次竖式经验 → 行动序列(写→换行→对齐→加→进位→回读)在经验流中共现
3. 共现波峰 (§1734) → AP自动发现这组行动频繁共现 → process_paradigm_binding
4. 范式绑成后, 条件是内生感受 (低把握→开始写) → 换5+8也能用
5. keyword_organization → "竖式"这个词成为范式锚
6. grammar_refinement → 对齐精度/进位时机优化

### 关键机制: 行动序列共现模式发现
当前 phase20_7 有 action_records (记录每个行动), 但没有"行动序列共现模式发现".
**需要新增**: 从 action_records 序列中, 发现频繁共现的行动子序列 (如"write_cell→move_focus→write_cell→read_draft"共现), 绑成可复用范式.

**这不增实体** — action_records 是既有表, 共现模式发现是从既有数据派生 (§132 派生可重建).
但发现机制本身是新的计算 (不是新表/新实体, 是新投影 — 同 _cognitive_feelings_from_pool 模式).

### 范式条件 = 内生感受 (§276 核心原理)
范式触发条件不是"看到3"而是:
- 低把握 (grasp低 → 不知道怎么算 → 开始写)
- 后继偏置弱 (写完一行 → 没有强波峰说接着写 → 换行)
- 数量感违和 (结果>9一位写不下 → 进位)
- 正确感低 (写完 → 感觉不对 → 回读验算)

这些条件全部是 §30 认知感受通道的值 — 已接通 (B-2路1+路2).
**范式 = 内生感受条件 → 行动序列**, 条件已有, 行动已有, 缺的是"绑定"机制.
