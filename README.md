## 这是什么

APV3.0 是 AP（Artificial PsyArch，人工心灵架构）的中文开放对话底座版本。它把一次交互变成可回放的 tick 闭环，现场展示中文开放对话、教学后实时学习、数学泛化、图像识图、画板外显、主动行为和记忆包导入/分享。

它以 tick 为周期运行完整认知闭环：

```
感受 → 状态池 → 短期结构池 → 召回(B) → 预测/追溯(C) → 认知感受 → 情绪 → 行动竞争 → 反馈 → 学习
```

教它什么它就学什么。没教过的它诚实说不会。它的"像人"是**过程像人**：会学、会回头看、会纠正、会把经验写回自己的记忆。

## 理论快读

### 一句话回答

AP（Artificial PsyArch，人工心灵架构）是一种白箱、持续、全栈拟人的认知架构。它以 tick 为周期运行，每一 tick 走完整闭环：感受 → 状态池 → 短期结构池 → 召回 → 预测/追溯 → 认知感受 → 情绪 → 行动竞争 → 反馈 → 学习。

### AP 和 Transformer 有什么不同

| 维度 | Transformer | AP |
|---|---|---|
| 架构 | 黑箱参数函数 | 白箱认知闭环 |
| 学习 | 海量数据预训练后冻结 | 小参数 + 持续在线学习 |
| 输出 | 一次性吐下一个 token | 多 tick 过程：观察 → 草稿 → 回读 → 修改 → 提交 |
| 泛化 | 模式匹配 | 经验后验 + 范式组合 |
| 可解释 | 事后热图 | 每 tick 审计槽可回放 |
| 情感 | 无 | 12 认知感受通道 + 情绪慢量 |

### 一个 tick 干了什么

```text
外界刺激 + 内源预测
  -> 感受器
  -> 状态池
  -> 短期结构池
  -> 召回 B
  -> 预测 C_forward / 追溯 C_backward
  -> C* 回灌
  -> 认知感受
  -> 情绪
  -> 行动竞争
  -> 反馈 + 奖惩
  -> 写入经验流
```

### 内生认知主义

认知过程本身产生的内部状态都会参与学习和行动竞争。AP 学的不是“看到 X 输出 Y”，而是“看到 X 后自己产生了什么感觉、召回了什么、该怎么做”。

### 视觉、听觉和内心画面

视觉和听觉都是一等公民 SA，进入同一套状态池和经验流。AP 展示出来的内心画面，不是原图缩略图，而是从状态池视觉 SA 重建出来的认知画面。焦点更清晰，周边更模糊，能回放它自己“怎么看图”。

### 召回 B / 预测 C / 追溯 C*

- B 负责找历史相似现状
- C_forward 负责“接下来会怎样”
- C_backward 负责“为什么现在这样”
- C* 把预测和解释一起回灌到状态池，参与下一轮竞争

### 12 认知感受和情绪

认知感受包括惊、违和、合理、正确、把握、期待、压力、闭合、未闭合、疲劳、节奏感、后果感。情绪是慢变量，会跨 tick 积分，调制表达松紧、探索保守和行动阈值。

### 为什么能数学泛化

数学能力来自“事实库 × 过程范式”：

1. 先教个位事实
2. 再教竖式过程范式
3. 未见题目由逐列子召回 + 进位链组合出来

因此 42+35、24+53 这类没见过的题可以被算出来，而没教过的事实缺口会直接诚实回退。

### 主观能动性

主动性不是定时器问候，而是未闭合张力和行动竞争的结果。张力累积到一定程度，AP 会在闲时自发外显、自己开口、自己回味。

### 行动竞争与 DraftGrid

AP 不是一次性整句输出，而是逐字写入草稿格、回读、修改、提交。每一轮候选动作都在竞争，输出是竞争胜者，不是模板硬吐。

### 范式发现与红线

APV3.0 允许范式从经验流里涌现，不靠答案表和关键词硬门。红线包括：

- 不用答案表
- 不用 hash 到 label 映射
- 不用学生侧 LLM 代答
- 不用 Python eval / 计算器
- 不用前端伪造 tick
- 不用 UI 直接改认知结果

### APV3.0 已达成

- 中文开放对话
- 教学后实时学习
- 视觉识图和内心画面重建
- 竖式数学泛化
- 主观能动性和闲时自发外显

## 常见质疑与回应

- Q: 它是不是规则引擎？ A: 能证明的是，它的主流程靠经验流、状态池、范式和行动竞争驱动，不靠手写规则把答案分支死锁住。
- Q: 它是不是 LLM 包壳？ A: 不是。LLM 只在教学协议里做外部教师、范式分析器和课程编译器，运行时 AP 自己做决策。
- Q: 数学泛化是不是答案表或计算器？ A: 不是。教过事实再教过程范式，AP 才能组合出没见过的题。
- Q: 画图和内心画面是不是复读最近一张图？ A: 不是。展示的是视觉记忆和草稿回放，条件不足时会诚实回退。
- Q: 主动性是不是定时器问候？ A: 不是。主动发言来自未闭合张力和行动竞争，不是固定周期。
- Q: 为什么不直接用 LLM？ A: 因为目标不同。LLM 是生成终点，AP 是每次交互都能学习的起点。
- Q: 这套东西能做什么？ A: 中文开放对话、可教学学习、数学泛化、图像识图、画板回放、主动行为、认知研究。

## 教学协议里的 LLM

AP 的教学系统把 LLM 用作**外部教师、范式分析器和课程编译器**：它负责分析样例、组织教学顺序、写入示范与反例。AP 的运行时决策链不经过 LLM，学习结果进入经验流后由 AP 自己竞争、回放和泛化。

教学函数 `teach_process_paradigm_demonstration` 可由 LLM 调用来高效灌入范式知识，让 AP 快速获得举一反三的能力。这条路径把 LLM 的占用压在教学阶段，推理阶段保持 AP 自主。
