什么是 AP？

AP（Artificial PsyArch，人工心智架构）是一种白箱、持续、全栈拟人的认知架构。它不是单个模型，也不是普通聊天机器人，而是一套让信息在多模态感受器、状态池、结构池、记忆、预测、追溯、认知感受、情绪、行动、反馈和自适应调参之间循环流动的心智运行框架。

AP 的核心是以 tick 为周期运行：每个 tick 中，外界刺激和内源预测共同进入状态池，形成认知场；被注意力调制的对象进入短期结构池形成认知流；召回历史经验，向后预测，向前追溯，形成 C* 预测/解释包回灌状态池；由此产生惊、违和、合理、正确、把握、期待、压力、闭合/未闭合等认知感受，再调制情绪与行动竞争；行动结果和奖惩反馈重新进入下一 tick，长期形成学习和适应。

内生认知主义

AP 的哲学核心是内生认知主义：认知过程本身产生的内部状态——惊、违和、合理、把握、期待、压力、未闭合、疲劳——都应作为可被感受、可被记忆、可被召回、可被学习的信号进入认知流程。这不是给系统加标签，而是让这些状态真正参与行动竞争和学习的闭环。

这意味着 AP 的"恐惧"来自对惩罚的预测（§27），"求知欲"来自认知压高伴随的惩罚信号（§27.1），"把握感"来自 B/C 召回的支持度和退火后验（§173.5）——全部从经验涌现，不由关键词设置。

为什么 AP 能数学泛化？

AP 的数学能力不是内置计算器（白皮书 §65.3 红线禁止 eval/solver），而是"事实库 × 过程范式"的乘法结构：

  1. 事实库（echo 阶段）：先教 AP 个位加法事实（2+3=5, 4+5=9...），AP 通过正常教学写入经验流。
  2. 过程范式（范式阶段）：教师示范竖式过程（先写第一行→换行对齐→逐列加→进位→回读验算），AP 从行动序列共现波峰自动发现这套范式。
  3. 组合：未教过的组合（如 42+35）由"逐列子召回+进位链"组合出来——每列结果是已教事实的 exact 召回，进位是"召回结果的高位进入下一列"。缺哪条事实就卡住，教了就通。

这正是儿童学习路径：先背加法表，再学竖式过程，然后能算没背过的题。

APV3 开放对话底座

APV3 的目标是多模态中文开放自由对话底座。它需要把 AP 的所有新理解落成真 runtime，不是继续修 demo。当前版本已实现：

  • ✅ 文本教学-召回-泛化-持久闭环
  • ✅ 视觉感受器+内心画面重建+教学共现
  • ✅ 12 认知感受通道+情绪慢量跨 turn 积分
  • ✅ 竖式数学过程范式（教事实+示范→未教组合逐列算出）
  • ✅ 行动竞争+DraftGrid 逐字写+回读+修改+提交
  • ✅ 范式发现（行动序列共现→涌现→偏置注入→反例抑制）
  • ✅ 感受 SA 回灌（AP"感到自己在慌"→元认知）
  • ✅ 自发外显（未闭合张力→idle 主动说"我还在想这个"）
  • ✅ 先天编码审计（InnateRule 表可查看 AP 的 DNA）

接口与使用

APV3 提供 Web API 和 CLI 两种接口：

  • POST /api/phase20_7/turn — 发送一轮对话（文本/图片/音频/教学反馈）
  • GET /api/phase20_7/turn... — 详见 web_chat.py 路由表
  • python -m apv3test.chat — CLI 命令行对话
  • python scripts/cold_retest_harness.py <db> — 冷重测召回通过率
  • python scripts/six_moments_acceptance.py — 六时刻自动验收
