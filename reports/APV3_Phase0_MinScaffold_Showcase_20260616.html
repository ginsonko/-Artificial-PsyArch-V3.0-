本轮结论

Phase0 最小地基已完成: APV3.0test 现在有独立包、接口契约、持久化契约、能量观察器、预测 ruler、runtime state codec、SQLite runtime/audit 分库包装与对应测试。

守住的 AP 边界

  • Bn 仍是“当前场像过去哪些经验”。
  • Cn 仍沿显式后继边读取“这个 B 之后通常发生什么”。
  • 持久化不是新策略, 只是同一套 AP-native 运行本体的磁盘落地与等价恢复。
  • audit db 可删, runtime ontology db 是唯一运行依赖。
  • 没有引入关键词路由、答案表、整句宏、学生侧 LLM 或数学字面答案加权。

本轮修复

  • 修复 SQLite 连接未关闭导致 Windows 上 audit db 无法删除的问题。
  • 修复显式关闭连接后缺少 commit 导致 runtime/audit 写入未落盘的问题。
  • 新增测试锁定“删除 audit db 后 runtime 仍可恢复”。
