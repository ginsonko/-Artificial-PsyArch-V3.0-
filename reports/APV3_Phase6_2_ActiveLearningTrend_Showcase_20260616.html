趋势样例一：失败推动求教

cue = goal::ask
context = ctx_work
failed ticks = 1, 20, 40

failure_count = 3
teacher_request = generated
teacher_request.energy.P = 1.0
teacher_request.energy.A = 1.0

含义：反复失败让求教压力进入状态池，但能量有上限，不会无限堆叠。

趋势样例二：成功让请求下降

失败两次: failure_count = 2
成功一次: failure_count = 1
再成功一次: failure_count = 0

request = None
suppressed_reason = below_request_threshold

含义：请求下降不是硬压，而是成功经验让失败 trace 自然回落。

趋势样例三：教师补习后 mastered

含义：LLM 可以作为老师加速，但学到的内容仍是 AP-native evidence，学生侧运行态不保存 LLM 策略。

失败样例与修复

首次 Phase6.2 目标测试失败，因为 observe() 使用浅拷贝，后一次成功观察会回写前一次结果里的嵌套 active_learning_failures。修复为 deepcopy(dict(state)) 后，历史 tick 的趋势记录可以独立审计。
