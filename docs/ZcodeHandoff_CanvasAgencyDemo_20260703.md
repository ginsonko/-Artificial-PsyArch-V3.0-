## T-A. 首页画板卡片

1. 首页加"画板"卡片（放内心画面下方或 tab 内）: HTML5 `<canvas>` 300×300，画笔（黑/红/蓝三色+
   粗细两档+清空按钮），鼠标/触摸可画。
2. **发送**: "把画发给 AP"按钮 → `canvas.toDataURL('image/png')` → 走既有
   `/api/phase20/media/upload` 拿 path → 作为 `image_path` 连同输入框文字（默认填"这是什么"）
   发一个 turn → 对话气泡里显示这张画（用户消息侧带图）。
3. 教学闭环引导: 卡片下方一行小字——"画个简单图形→问它是什么→它不认识时教它→再画类似的试试"。
4. **AP 草稿的图片化外显**（用户点名"画好的内容和草稿一样以图片形式发送到对话框"）:
   AP 回复气泡下加"查看草稿过程"小按钮 → 弹出用 tick trace `draft_grid.cells` 渲染的二维网格
   （每格一字符，按 tick 淡入重放写字过程），可一键"存为图片"（html2canvas 或 canvas 重绘均可）。
   数学竖式 turn 的逐列过程也由此可见（用户此前点名"看不到列竖式过程"——这里一并解决）。
- 验收: 画圆→发送→教"是圆圈"→再画圆再问→答对；AP 草稿网格重放可看、可存图。

## T-B. 首页"主观能动性演示"按钮

与"小学数学泛化学习示例"并排，名称"主观能动性演示"。点击后自动执行（每步间隔让用户看清，
消息区同步系统提示解说）:
1. 系统提示:"先教 AP 一句惦记的表达" → turn: 教"明天吃什么好呢"→反馈"我还在想这个问题"(reward)
2. 系统提示:"现在问它一个它答不上的问题，连问三遍，不给答案——张力会累积" → 三个 turn"周末去哪玩好呢"
3. 系统提示:"现在什么都不做，开启闲时心跳，看它自己会不会开口" → 自动开 auto-idle
4. AP 自发说出"我还在想这个。"（spontaneous 气泡）→ 系统提示:"它主动提起了没解决的事——
   这不是定时器，是未闭合张力驱动的自发行为。再等等看，它说过一次就不会烦你了。"
5. 下一个 idle 安静后 → 系统提示:"演示结束。它知道分寸。"
- **红线**: 全部走真实 turn/idle；第 4 步如果 AP 没说话（张力衰减时序问题），多等 2-3 个 idle，
  仍不说则显示"这次它没开口——张力是真实变量，可再问一遍第 2 步的问题加压"，**禁止伪造气泡**。
- 验收: 完整跑通一次演示且气泡真实；连点两次按钮不产生错乱（第二次前自动换 session 或提示）。
