APV3.0test Phase7.1 不能决共享片段

本阶段修复 multi-reply 共享 Cn 被机械填入多个槽的问题：共享片段可以进入草稿/想法，但不能自动提交成确定回答。

设计语义

only shared fragment recalled
  -> may enter draft/thought
  -> must not fill all unresolved slots
  -> must not auto-commit as decided reply

落地变化

模块变化
ParadigmSlotFiller记录未解决槽；后续锚点/共享片段标记 undecidable_fragment。
MinimalDialogueRuntime含不能决片段时不自动 commit。
IncrementalTickRuntimeteacher-off Cn successor 不再被伪装成普通槽候选。

样例

train:
  三 顾 -> 茅 庐
  三 顾 -> 草 庐 之 中

teacher-off:
  cue = 三 顾

result:
  Cn successor = 庐
  emitted = 庐
  committed_text = ""
  undecidable_fragment = true

保护门

train:
  你 好 -> 我 在

teacher-off:
  cue = 你 好

result:
  emitted = 我 在
  committed_text = 我在
  undecidable_fragment = false

边界

可以宣称：最小“不能决片段”处理能力成立，多义共享片段不再重复填槽或自动提交。

仍不能宣称：系统已经能自然表达“茅庐还是草庐之中?”，也还不能主动澄清。
