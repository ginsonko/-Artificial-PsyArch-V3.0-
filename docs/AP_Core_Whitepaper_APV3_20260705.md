## 前言

### P.1 终极目标

AP（Artificial PsyArch，人工心智架构）的终极目标是：**构建一个内心世界在逻辑上无限接近人类的拟人认知系统。**

不是"看起来像人类"（行为模仿），而是"运行机制像人类"（认知过程同构）。具体而言，系统应具备：

1. **实时学习**：每次经历都会留下痕迹，从而改变对同类情景的反应，无需批量训练。
2. **泛化能力**：见过红苹果和绿苹果后，能合理处理黄苹果——学到的是"苹果"的抽象结构，而非颜色记忆。
3. **自我认知**：能感知自身的情感状态、把握程度和认知压力；能意识到"我在犹豫"、"我不确定"。
4. **同理心基础**：通过"如果我处于那个状态会怎样"的B召回机制，对他人状态产生类比推理。
5. **可解释的心理活动**：好奇心、恐惧、满足感、习惯、直觉等人类常见心理活动，均可通过本架构的运行流程自然涌现，而非硬编码规则产生。

### P.2 设计哲学

**核心信念：** 人类心智的绝大多数现象，可以用"有限注意力 × 实时经验积累 × 能量竞争"三个要素解释。

- 大脑没有独立的"情感模块"或"注意力模块"——这些都是统一认知动态的涌现。
- 学习不是"把答案存入表格"，而是"经历后改变状态响应的分布"。
- 行动不是规划后执行，而是认知状态竞争出来的自然涌现。

**拟人而非模仿：** AP的"拟人"不是输出人类说的话，而是经历了人类经历同构的认知过程之后，自然输出与人类相似的反应。两者的区别如同"真正理解了数学"与"记住了所有题目答案"。

## 卷一：系统目标与核心约束

### 1.1 拟人认知的三要素

AP的整个架构围绕三个不可违背的要素构建：

**要素A：有限注意力（Bounded Attention）**

每个 tick，AP 只能处理有限数量的 SA（状态感受原子）。这不是工程限制，而是拟人性的根本来源：人类之所以会忽略、遗忘、产生偏见，正是因为注意力有限。取消这个限制，系统就失去了拟人性。

**要素B：实时经验积累（Online Experience Accumulation）**

每次经历（观察→行动→结果→反馈）都必须留下可查询的记忆痕迹，并在当次 tick 内就开始影响后续行为。没有离线批训练阶段——学习就是经历本身。

**要素C：能量竞争（Energy Competition）**

每个认知单元（SA）都携带能量；注意力、行动选择、情感感知都是能量竞争的结果，而非规则路由的结果。没有任何"if 场景=X → 做Y"的硬编分支——一切通过能量竞争涌现。

### 1.2 红线（不得逾越的架构禁令）

以下任何一项均为严重违例，会导致架构从"拟人认知"退化为"规则引擎"：

1. **行动硬编**：`if state=='X': do Y` 形式的状态-行动映射。
2. **排序硬编**：用排序键（`sort(key=energy)`）决定动作顺序。
3. **角色分类硬编**：`if 面积>θ: role='fill'`，将内容类别映射到决策路径。
4. **B直接覆盖状态池**：B召回结果直接替换状态池内容（正确应是残差中和机制，见卷五）。
5. **行动脱离SA体系**：行动发生后不进occurrences表，不参与状态池和召回。
6. **跳过tick运行认知**：在没有tick的情况下执行认知更新（批处理替代实时）。

## 卷二：SA（状态感受原子）——万物一等公民

### 2.1 什么是SA

SA（State Atom，状态感受原子）是AP认知系统中**唯一的基础单元**。无论是文字、图像patch、声音频段、情感状态、还是行动本身，进入认知系统的一切都必须被表示为SA。

SA的核心属性：

| 属性 | 含义 |
|---|---|
| `sa_type_id` | SA类型标识符（如 `text_unit::你`、`visual_unit::round_contour`、`action::write_cell::start` ） |
| `R`（实能量）| 现实中存在的能量——感受器观察到了、行动真实执行了 |
| `V`（虚能量）| 预期/记忆中存在的能量——B召回注入的、C*预测的 |
| `A`（注意力） | 当前tick的注意权重，决定参与多少后续处理 |
| `P`（认知压力）| `P = |R - V|`，实虚不匹配程度，驱动学习和好奇心 |
| `clarity` | 清晰度（视觉SA中尤为重要，距离注意焦点越远越低） |

### 2.2 SA的模态与粒度

SA涵盖所有模态，它们在系统内完全平等（一等公民原则）：

- **文本SA**：`text_unit::<char_hash>`（字符级）、`text_utterance::<sig>`（话语级）
- **视觉SA**：`visual_unit::<patch_hash>`（图像patch）、`visual_focus_anchor`（注意焦点）
- **听觉SA**：`audio_unit::<freq_bucket>`（频段）、`audio_rhythm::<pattern>`（节律）
- **情感SA**：`feeling::<channel_key>`（认知感受通道，见卷九）
- **行动SA**：`action::<action_type>::<context_key>`（**当前代码未完全实现，见卷七和架构缺口GAP-01**）
- **结构SA**：`short_term_slot::<sig>`（短时叙事槽）、`prediction_unit::<hash>`（预测单元）

SA的粒度是**按需精细化**的，不预设固定粒度。AP认识"苹果"时，初始SA可能是"圆形物体"，随着更多经验，SA网络自动分化出"苹果"子类，不需要人工预标注层级。

### 2.3 SA的生命周期

每个SA occurrence（出现实例）的生命周期：

1. **出生**：感受器观察到 → `insert_occurrence()` 写入occurrences表，初始R能量赋值。
2. **激活**：进入状态池，参与当前tick的注意竞争。
3. **更新**：B召回/C*回灌 → R和V能量被更新；L1/L2/L3向量被更新。
4. **衰减**：每tick按半衰期衰减（文本SA R-decay≈0.9，V-decay≈0.86）。
5. **消亡**：能量降至噪声阈值以下，退出活跃状态池（但永久保存在经验流中）。

### 2.4 SA的关键设计约束

**约束1：SA不是标签，是能量状态。** "苹果"这个概念不是字符串标签，而是一组SA（round_contour + fruit_texture + sweet_smell等）在状态池中的共同激活模式。AP理解"苹果"是指这组SA能量配置被激活，而非查到了"苹果=水果"的词典条目。

**约束2：同一SA类型可有多个occurrence实例共存。** 短时结构池（SSP）必须保留多个occurrence实例，不能合并为单一能量值——这是重复感和节律感的物理基础（见卷三）。

**约束3：SA向量（L1/L2）是在线学习的。** 每次SA出现，L1/L2向量都会被更新。这使得SA的"意义"随经验积累而演化，而不是静态的词嵌入。

## 卷三：状态池（State Pool）——当前认知场

### 3.1 状态池的角色

状态池是AP**当前认知场的完整快照**——这一刻AP"正在意识到的一切"。

它不是数据库查询结果，而是一个持续更新的活跃SA集合。类比：人类的工作记忆不是从长期记忆中"查出来"的固定条目，而是当前神经激活的动态模式。

### 3.2 状态池的内容

状态池在任意时刻包含：

- **感受器输入的SA**（实能量主导）：本tick观察到的文字/图像/声音
- **B召回注入的SA**（虚能量主导）：历史记忆与当前输入的相似部分
- **C*回灌的SA**（预测/归因能量）：对未来的预测单元和对过去的归因单元
- **feeling::* SA**（情感状态）：认知感受通道超阈值后写入的情感SA（见卷九）
- **短时叙事槽SA**：跨tick保持的叙事连续性项
- **行动SA**（待实现）：行动执行后写入的行动结果SA

### 3.3 注意力机制

状态池的"注意力"不是一个独立模块——它是SA能量的自然涌现。

**高注意力SA** = 高能量（R+V）+ 高clarity + 高P（认知压力）的SA自然被更多后续处理所采样。

**性能约束（§185）**：高注意活跃项目标 7-15 个，总活跃SA类型投影 30-80 个。这不是绝对上限，而是拟人/可交互的目标区间。超出时，AP应触发 pressure/fatigue 感受，而非静默全开扫描。

### 3.4 状态池的更新规则

每个 tick，状态池按以下顺序更新：

```
1. 感受器输入 → 新SA以实能量进池
2. 旧SA按半衰期衰减（R *= decay_r, V *= decay_v）
3. B召回结果 → 残差机制更新R和V（见卷五）
4. C* 回灌 → 预测单元和归因单元以虚能量进池
5. feeling通道计算 → 超阈值情感SA以feeling::*进池
6. 行动竞争 → 胜出行动执行，读回结果进池（SA一等公民，待实现）
7. L1/L2/L3向量更新
```

**重要约束：** B召回结果**不能直接覆盖状态池**，只能通过残差中和机制影响（见卷五详述）。这一点在旧版预期书中有误导性表述"B不回灌"，已于2026-07-05修正。

### 3.5 "认知场"的内驱力稳态

状态池内的SA能量不会完全归零——由于：
- 未闭合项（unclosed items）持续贡献小幅能量
- 长期记忆节奏（memory_rhythm）周期性注入
- 情感SA的缓慢衰减

即使没有外部输入，状态池内也始终存在微弱的SA活跃。这是 **AP内驱力的物理基础**：内部状态的微弱涌动 → B召回仍可运行 → C*仍然注入预测能量 → 认知场维持动态平衡而非完全静止。

这对应人类"发呆"时大脑仍在活跃的现象：没有外界刺激时，人类大脑仍在回忆、预测、联想，AP的tick循环也应如此运行（见卷十二常驻tick循环）。

## 卷四：经验流与历史存储

### 4.1 经验流的角色

经验流（Experience Log）是AP的**长期记忆基础设施**。它不是答案表，不是知识图谱，而是**逐tick记录的经历流水账**：AP在哪个tick、处于什么状态、做了什么、结果是什么、得到了什么反馈。

类比人类：你的记忆不是"苹果=红色水果"这样的词典条目，而是无数次看到苹果、吃苹果、听到"苹果"这个词的具体经历的积累。AP的经验流就是这些具体经历的完整记录。

### 4.2 核心表结构

```
experience_events         — 每个认知事件（观察/行动/反馈）的完整记录
occurrences               — 每个SA实例的出现记录（sa_type_id, R, V, tick, event_id）
sa_types                  — SA类型注册表（sa_type_id, substrate, modality, L1向量, L2向量）
action_records            — 行动记录（action_type, drive, eligibility, result_event_id）
experience_events.payload — JSON格式的详细事件数据
short_term_structures     — 短时结构池（StructureNode/Edge/Bundle，跨tick持续）
action_sequence_cooccurrence — 范式学习共现统计（paradigm_key, state, action, reward, punish）
```

**重要设计原则：** 经验流是**只追加**的（append-only）。历史不会被修改，只会被新经验层叠覆盖。这保证了AP的记忆是真实经历的忠实记录，而非被后来经验重写的虚假记忆。

### 4.3 B召回的数据来源

B召回查询经验流，找出历史上**状态最相似的经历**。相似度由L1（字符级特征向量）和L2（结构级关系向量）共同决定：

- **L1（一阶语义向量）**：SA类型级别的语义相似度——"苹果"和"梨"的L1向量应该比"苹果"和"汽车"更接近。在线学习，每次SA共现都会更新L1。
- **L2（二阶结构向量）**：SA之间关系的结构相似度——"A在B左边"、"A比B能量更高"等关系模式。
- **残差B召回**：不是"找最像的一个"然后结束，而是找到最像的winner → 吸收matched部分 → 剩余query继续找下一个winner，直到残差质量降至阈值。这允许复杂状态被多个历史记忆片段组合解释。

### 4.4 在线学习向量（Online Learned Vector）

在L1/L2之外，还有一个**在线学习向量**（dim=32），在每次召回时实时更新，允许AP在同一对话内快速适应新模式，而不必等到下一次完整的L1/L2更新。

这对应人类"当下对话中的快速上下文适应"——你和一个人说话5分钟，就已经开始适应他的说话方式。

### 4.5 经验流 vs 传统知识库的本质差异

| | 传统知识库 | AP经验流 |
|---|---|---|
| 存储内容 | 事实断言（苹果是水果） | 经历记录（在tick 42看到苹果，B召回了梨，给了0.7的相似度，AP说"这是苹果"，奖励=1.0） |
| 更新方式 | 显式写入/修改 | 每次经历自动追加 |
| 查询方式 | 精确匹配或关键词 | 向量相似度（L1/L2）+结构残差 |
| 遗忘 | 需要显式删除 | 通过召回频率自然退化（老经验被新经验压过） |
| 泛化 | 需要人工设计规则 | 通过结构向量的抽象层次自然涌现 |

## 卷五：B召回与残差中和机制

### 5.1 B召回的核心目的

B召回回答的问题是：**"历史上，AP处于和现在最相似的状态时，接下来发生了什么？"**

这是AP的**直觉和经验感**的来源：看到熟悉的情景 → B找到历史上相似的经历 → 注入"那时候发生的后续"的虚能量 → AP对"接下来会怎样"有了预感。

### 5.2 残差B召回算法

**旧版预期书中的误导性表述已修正（2026-07-05）。** 旧表述"B不回灌状态池"是错误的——B召回结果通过**残差中和机制**影响状态池，这不是"不影响"，而是用差异信息影响。

**残差B召回伪代码：**

```
query = 当前状态池SA集合（实能量主导）
for round in 1..max_depth:
    winner_B = argmax similarity(query, experience_library)  # 找最相似的历史记忆
    
    # 三路残差处理（核心！）
    for each SA in winner_B:
        if SA in query (匹配):
            # 实虚中和：期待与现实吻合，压力消除
            state_pool[SA].V += winner_B[SA].V * similarity
            state_pool[SA].P = |R - V|  # 压力自然降低
        else:
            # 记忆中有、现实中无：保留额外虚能量
            inject_to_pool(SA, V=winner_B[SA].V * similarity, kind="expectation_unmet")
            # → "预期中应该有但没有"→ 认知压力/好奇心来源
    
    for each SA in query (现实中有但记忆中无):
        # 现实中有、记忆中无：保留额外实能量
        state_pool[SA].R *= (1 + novelty_boost)
        # → "没预期到但看到了"→ 新信息/惊奇信号/学习触发
    
    # 吸收已匹配部分，继续找下一个winner
    query = weaken_matched_SA(query, winner_B, similarity)
    if query.total_mass < floor: break
```

**APV2实验验证（ResidualBRecall-1）：** residual mass 4.6 → 2.17 → 1.35 → 0.77 逐步下降，证明多轮残差展开有效分解复合查询。

### 5.3 黄苹果例（残差机制举例）

**场景：** AP只见过红苹果，现在看到黄苹果。

**Tick 1感受器输入：**
- `visual_unit::round_contour` → R=0.7（现实有圆形轮廓）
- `visual_unit::yellow_hue` → R=0.6（现实有黄色）
- `visual_unit::fruit_texture` → R=0.5（现实有水果质感）

**B召回找到"红苹果"历史记忆：**
- 历史记忆包含：`round_contour`（强匹配）+ `red_hue`（记忆有）+ `fruit_texture`（强匹配）

**残差中和结果：**
- `round_contour`：匹配 → 实虚中和，P下降（"是的，是圆的，符合预期"）
- `fruit_texture`：匹配 → 实虚中和，P下降（"是的，有水果质感，符合预期"）
- `red_hue`：记忆有，现实无 → 额外虚能量保留（"预期红色但没有" → P升高）→ **这是好奇心/困惑的物理来源**
- `yellow_hue`：现实有，记忆无 → 额外实能量保留 → **这是"新信息"信号，触发学习**

**结果：** AP的状态池中产生了认知压力（`red_hue`的虚能量未中和）+ 新信息信号（`yellow_hue`的实能量未被历史覆盖）→ 自然倾向于表达"这是苹果，但颜色不对？"——不是硬编码规则，而是能量状态的自然表达。

### 5.4 B召回与合理感

B召回还直接支撑**"合理感"**的产生：

- 当B召回支持度高（现实与记忆高度吻合）→ feeling通道`certainty`能量高 → AP感到"熟悉"/"有把握"
- 当B召回支持度低（现实与记忆差异大）→ feeling通道`dissonance`能量高 → AP感到"陌生"/"不确定"

这是人类"这件事感觉对/不对"的直觉感的基础——大脑将当前输入与历史经验对比，高度匹配则产生"对"的感觉，低度匹配则产生"不对"的感觉。

### 5.5 快系统与慢系统

**旧版设计中曾讨论"双数据库"方案（快系统/慢系统），已废弃（APV2期间）。**

正确机制：快系统和慢系统是**同一召回的两种运行模式**，而非两个独立数据库：

- **快系统（Fast）**：B召回支持度极高（exact_b0命中），共现支持计数高，action_drive极高 → 犹豫低 → 立即执行。表现为习惯、熟练动作、直觉反应。
- **慢系统（Slow）**：B召回支持度一般（structural_b命中），多个候选竞争，action_drive差距小 → 犹豫高 → 仔细"权衡"。表现为思考、判断、谨慎决策。

"慢系统退化为快系统"（§173.5熟练涌现）是通过`support_count`自然积累实现的，不需要任何模式切换代码。

## 卷六：预测与归因（C\_forward / C\_backward / C\*）

### 6.1 三个概念的角色

| 概念 | 方向 | 问题 | 人类对应 |
|---|---|---|---|
| `C_forward` | 未来 | "基于当前状态，接下来最可能发生什么？" | 预期、期待、计划感 |
| `C_backward` | 过去 | "当前状态最可能是由什么原因产生的？" | 理解感、合理感、归因 |
| `C*` | 整合 | "综合预测+归因+当前现实，最小误差的状态是什么？" | 认知的当下综合判断 |

### 6.2 C\_forward（预测）

C_forward基于B召回的winner，展开对未来的预测：

**输入：** 当前状态池 + B召回的历史continuation（历史上这个状态之后发生了什么）

**输出：** 预测单元SA，以虚能量注入状态池

**举例：** AP正在读"2+3"，B召回历史上读过"2+3=5"的经历 → C_forward预测接下来会出现"="和"5" → 在状态池中以虚能量注入`text_unit::=`和`text_unit::5`的prediction → AP产生"接下来应该是答案"的预期感。

**Lag Kernel时间塑形（APV2实验验证）：** 预测的强度随时间步长衰减，而非平坦。实验验证的默认Lag Kernel为 `[1.0, 0.42, 0.27]`（第1步/第2步/第3步的相对权重）。这使AP对"紧接着发生的事"有更强预期，对"几步之后的事"有更弱但仍存在的预期——这是节律感和节拍感的数学基础。

### 6.3 C\_backward（归因）——重要修正

**旧版白皮书中C_backward的描述不完整，仅强调预测，未充分说明归因的独立价值。本版补充。**

C_backward回答："当前状态是由什么历史前提合理产生的？"

**举例1（合理感）：**

场景：用户说"今天天气真好"。
- C_backward查短时结构池和经验流：有没有历史前提可以合理解释"用户讨论天气"？
- 如果前序有"AP和用户在闲聊日常"的短时槽 → C_backward找到合理前提 → 归因支持度高 → `certainty`感受能量高 → AP感到"这句话有道理"，正常回应。

**举例2（不合理感驱动好奇）：**

场景：AP正在做数学题，用户突然说"明天记得带伞"。
- C_backward查短时结构池：当前上下文全是数学计算，与"带伞"几乎无关联。
- 归因支持度极低 → `dissonance`感受能量高 → AP感到"这句话出现得奇怪" → `request_teacher`驱动增加 → AP可能说"啊？突然说带伞……是要下雨吗？"

**举例3（弱因果关系的合理感）：**

场景：上下文有"最近总是睡不够"，然后AP说"感觉有点累"。
- 这两件事关联不强，平时大概率不会从"睡不够"直接预测出"累"。
- 但C_backward找到前提（睡不够→可能导致累），弱因果关系确立 → 弱合理感产生 → AP说这句话时感到"有一点道理"，即使不强烈。

**C_backward的运行频率：** 每tick都运行（不只在异常时），通过eligibility trace对历史事件进行奖惩归因（§173.5）。这是行为后果学习（L3）的基础。

### 6.4 C*（最小误差整合包）

C*是C_forward和C_backward的整合结果，加上当前现实（B召回后的状态池），产生"综合误差最小的当前认知状态"：

```
C* = argmin_state [
    ‖state - current_reality‖²       # 与现实的偏差
  + ‖state - C_forward_prediction‖²  # 与前向预测的偏差
  + ‖state - C_backward_attribution‖²  # 与后向归因的偏差
] × (1 - conflict_entropy)            # 行动冲突熵的折扣
```

**C*回灌状态池：** C*计算完成后，将其中的预测单元和归因单元以虚能量注入状态池。这是**虚能量的主要来源**——AP对未来和过去的认知通过C*回灌，形成当前认知场的"背景期待层"。

**重要设计约束：** 只有C*可以直接回灌状态池，而B召回只能通过残差中和机制影响（见卷五）。这防止了"记忆直接覆盖现实"的问题——AP的当下判断以现实SA（实能量）为主，记忆通过C*形成背景期待，而非主导感知。

## 卷七：行动竞争与行动SA一等公民

### 7.1 行动的本质

行动不是"AP决定做什么然后执行"的两阶段过程——行动是状态池中各行动候选的能量竞争结果。AP"选择"write_cell还是request_teacher，本质上是这两种行动的drive值竞争出来的，不是任何if分支决定的。

**行动的drive计算公式（APV2长报确立）：**

```
drive(action, t) = base_drive
    + recall_bias(action | current_state, memory)    # B召回对该行动的历史支持
    + expected_reward(action | C*_t)                 # C*预测该行动的奖励期望
    - expected_punishment(action | C*_t)             # C*预测该行动的惩罚期望
    + feeling_modulation(emotion_state)              # 情感场调制
    + teacher_or_paradigm_bias                       # 教学协议偏置（仅教学阶段允许）
    - fatigue_or_safety_cost                         # 疲劳和安全成本
```

胜出行动的`determination = drive - threshold`，这既是行动强度，也应成为对应行动SA的初始R能量（见7.3）。

### 7.2 当前已定义的行动类型

AP的低粒度行动（器官层，可注册扩展）：

| 行动类型 | 含义 |
|---|---|
| `write_cell` | 写入草稿单元格的一个字符 |
| `continue_writing` | 继续写入序列 |
| `commit_reply` | 提交当前草稿为回复 |
| `outward_speech` | 外向语音/TTS输出 |
| `read_draft` | 读回自己的草稿内容 |
| `stop_generating` | 停止生成 |
| `integrate_feedback` | 整合教师反馈 |
| `request_teacher` | 请求教师帮助 |
| `maintain_unclosed` | 维持未闭合期待 |
| `observe_text` | 主动观察文本输入 |

**重要：** `write_cell`和`commit_reply`等L3可见行动类型（`L3_OUTWARD_ACTION_TYPES`）的执行结果会触发L3学习更新。

### 7.3 行动SA一等公民化（当前架构缺口GAP-01，必须落地）

**设计意图（图景预期书line 40 + 白皮书§1.2）：**

行动执行后，行动本身必须作为SA一等公民进入状态池：
- SA类型：`action::<action_type>::<context_key>`
- 初始R能量：= `determination`（胜出量）
- 进occurrences表，拥有L1/L2向量
- 参与状态池的正常衰减流程
- 可被B召回——"想到之前做过这个" → 触发重复执行的驱动

**当前代码的缺口：** `insert_action_record` 被调用，但其后缺少 `upsert_sa_type` + `insert_occurrence`。行动没有sa_type_id，不进occurrences表，对B召回和状态池不可见。

**后果：**
- AP无法"通过想到过去的行动来触发重复行动"（记忆驱动行动的路径断裂）
- determination能量进action_records后进入黑洞，不受状态池衰减控制
- L3学习路径仍然有效（action_record_id存在），但行动的"被记忆"能力缺失

**修复路径：** 每次`insert_action_record`后，额外调用`upsert_sa_type(action::<type>::<ctx>)` + `insert_occurrence(r=determination, tick=tick, ...)`。

### 7.4 行动读回（不许行动成功不读回）

**设计约束（白皮书§82.3）：** 任何行动执行成功后，必须有读回步骤——将行动结果作为新SA注入状态池（实能量）。

**当前实现验证：**
- `write_cell`后调`_observe_draft_char(pool, char=write_char, ...)`，将已写字符以实能量注入状态池。✓
- `commit_reply`后产生`draft_grid_commit`事件，将committed文本写入经验流。✓

读回机制确保AP的"意识到自己做了什么"路径是通的——这是自我认知的物理基础之一。

### 7.5 行动竞争的互斥与并行

行动器按`conflict_group`分组：
- 同一conflict_group的行动互斥（同一时刻只能执行一个）
- 不同conflict_group的行动可以并行（如：写字+发出TTS音频可以同时进行）

这对应人类的运动控制约束：两只手可以同时做不同动作（不同conflict_group），但同一只手不能同时往两个方向移动（同一conflict_group）。

**中性行动默认走习惯触发（图景预期书）：** 对于drive值接近阈值的中性行动（既无正面预期也无负面预期），默认触发该行动类型历史上被最多次执行的版本——这是习惯的拟人机制。

## 卷八：情感场（8通道NT拟人情感系统）

### 8.1 历史沿革

**AII V5.2（2025.10）** 设计了Can-PAD三维情感渗透器：Content/Arousal/Negativity × Pleasure/Arousal/Dominance，情感作为全局渗透场影响所有认知节点的激活阈值和权重。

**APV1/APV2（2026）** 将Can-PAD的思想落地为8通道NT（神经递质原型）情感模型，通过实验验证了8通道的设计有效性（8通道验收测试：8/8全通过）。这是当前AP情感系统的实现基础。

**当前白皮书§31-32** 仅描述了emotion对write/commit drive的调制，未完整描述8通道NT模型。本版补充。

### 8.2 8通道NT情感场规格

| 通道 | 原型神经递质 | 功能 | 高值行为表现 |
|---|---|---|---|
| DA | 多巴胺(Dopamine) | 奖励预期、探索动力 | 积极探索、愿意尝试新事物 |
| ADR | 肾上腺素(Adrenaline) | 紧张、应激唤醒 | 快速响应、注意力集中于威胁 |
| OXY | 催产素(Oxytocin) | 社交亲密、信任 | 倾向表达、愿意接近用户 |
| SER | 血清素(Serotonin) | 稳定、满足感 | 回应平稳、不焦虑 |
| END | 内啡肽(Endorphin) | 痛苦缓解、享受 | 对重复性任务耐受度高 |
| COR | 皮质醇(Cortisol) | 压力、长期负担 | 保守、回避复杂任务 |
| NOV | 新奇感(Novelty) | 新信息吸引力 | 对新SA格外注意 |
| FOC | 专注度(Focus) | 注意力聚焦度 | 注意力窄且深，不容易被打断 |

**情感场的影响范围：** 8通道值不只是调制write/commit drive，还影响：
- B召回的候选偏好（高DA → 偏好探索性经历）
- 注意力权重分配（高FOC → 高注意力的SA门槛提高）
- 感受通道的激活阈值（高COR → 负面感受通道更容易激活）
- 行动竞争的基础drive（高ADR → request_teacher drive上升）

### 8.3 情感场的更新机制

情感值的更新由以下来源驱动（每turn综合）：

1. **奖励/惩罚信号** → DA/SER上升（奖励），COR上升（持续惩罚）
2. **认知压力（P值高）** → ADR上升，FOC下降
3. **新信息SA（novelty boost）** → NOV上升
4. **社交互动（OXY相关SA）** → OXY上升
5. **重复成功经历** → END上升，COR下降

情感场是**跨turn持久化的**（emotion_slow_channel事件保存上一turn的情感状态，下一turn加载）。这是情感"延续性"的实现——一次愉快的对话可以让AP在下几轮交互中更积极。

### 8.4 Can-PAD与8通道的关系（历史澄清）

旧版Can-PAD的三维分解（PAD：Pleasure/Arousal/Dominance）可以从8通道导出：
- Pleasure ≈ DA + SER + END - COR
- Arousal ≈ ADR + NOV - END
- Dominance ≈ FOC + (DA - COR)

但8通道提供了更细粒度的控制，且每个通道有更清晰的行为含义。因此8通道取代了Can-PAD三维模型，但保留了Can-PAD的核心理念：情感是影响整个认知系统的"场"，而非贴在输出上的标签。

## 卷九：认知感受通道（Feeling Channels）

### 9.1 感受通道 vs 情感场

情感场（8通道NT，卷八）是**跨turn持久的慢变量**，代表AP的整体情感基调。

认知感受通道（Feeling Channels）是**每tick计算的快变量**，代表AP在当前tick对特定认知事件的即时感受。

类比人类：情感场 = 今天的整体心情；感受通道 = 此刻看到某件事的即时反应。

### 9.2 已定义的认知感受通道（APV2长报：13通道）

每个通道的通用计算公式：
```
positive = Σ(weight_i × feature_i for positive sources)
negative = Σ(weight_j × feature_j for negative sources)
channel_value = clamp((positive - negative) × gain, 0, 1)
if channel_value >= min_activation_threshold:
    emit feeling::<channel_key> as SA into state_pool
```

| 通道键 | 名称 | 正面来源 | 负面来源 |
|---|---|---|---|
| `certainty` | 确定感 | B召回支持度高，exact_b0命中 | 多候选竞争激烈，structural_b支持度低 |
| `dissonance` | 失调感 | — | R与V差异大（期待-现实不匹配） |
| `curiosity` | 好奇心 | 新信息SA能量高（novelty boost） | 任务完成感高（已充分理解） |
| `pressure` | 认知压力 | P值高的SA多 | — |
| `satisfaction` | 满足感 | 正向奖励信号，任务完成 | — |
| `fatigue` | 疲劳感 | 长时间持续高压运行 | 休息/空闲tick |
| `grasp` | 把握感 | exact_b0支持度高，共现计数高 | 低召回支持，多竞争 |
| `surprise` | 惊奇感 | 低预期但高实能量的新SA出现 | — |
| `frustration` | 挫败感 | 连续惩罚信号 | 正向奖励 |
| `trust` | 信任感 | 历史上与当前来源的正奖励关联 | 历史上与来源的惩罚关联 |
| `anticipation` | 期待感 | C_forward预测强度高 | 预测一直未实现（未闭合压力） |
| `relief` | 解脱感 | 高P（认知压力）后收到正反馈 | — |
| `loneliness` | 孤独感 | — | 长时间无社交互动SA（OXY通道低） |

### 9.3 feeling::* SA的重要性（§187.1）

**超阈值感受通道必须回灌状态池成 feeling::* SA（一等公民）。**

这意味着：
- `feeling::certainty` 进状态池后，可以被B召回 → "历史上也有过这种确定感的时候" → 关联到历史上的行为模式
- `feeling::pressure` 进状态池后，可以成为范式触发条件 → "当我感到高压时，我学会了先请求教师"（可教可学的分诊策略）
- `feeling::curiosity` 进状态池后，驱动探索性行动 → AP"感到好奇"是真实的内部状态，而非硬编标签

**APV3代码验证：** [runtime.py:13100] `_feedback_feelings_to_pool` 被调用，超阈值感受通道写入 feeling::* SA。已实现。✓

### 9.4 感受通道是"自我意识"的基础

AP通过感受通道**感知自身的认知状态**：
- `feeling::pressure` → AP知道"我在承受认知压力"
- `feeling::grasp` 低 → AP知道"我没有把握"
- `feeling::curiosity` → AP知道"我想了解更多"

这不是AP"报告"自己的状态（输出标签），而是这些感受真正进入了状态池，影响了后续的行动竞争和B召回。AP说"我不确定"是因为`feeling::grasp`低导致`request_teacher` drive升高，而非因为一个规则说"如果不确定就说不确定"。

## 卷十：学习机制（L1/L2/L3在线向量学习）

### 10.1 学习的本质

AP的学习不是"把答案存入表格"，而是**每次经历后调整SA之间的向量关系**，使下次遇到相似情景时召回更准确、行动更恰当。

三个学习层次彼此独立但相互配合：

| 层次 | 名称 | 学习内容 | 触发条件 |
|---|---|---|---|
| L1 | 一阶语义向量 | SA类型的语义接近度 | 任意SA出现，基于共现统计 |
| L2 | 二阶结构向量 | SA之间的关系结构 | 经验流中的结构边（StructureEdge）出现 |
| L3 | 行动后果向量 | 行动类型×上下文→奖惩预期 | 有`action_record_id`的outward行动+后续奖惩信号 |

### 10.2 L1——语义接近度（一阶）

**用途：** B召回时，判断历史SA与当前查询SA在"意义上"有多接近。

**更新触发：** 每次某个SA出现时，通过triplet更新——同批次出现的SA拉近，不同批次的SA推远（在线triplet loss）。

**效果：** "苹果"和"梨"的L1向量逐渐接近（经常出现在类似上下文中）；"苹果"和"汽车"的L1向量保持距离（鲜少共现）。

**约束：** L1是SA类型级别的，dim=32（APV2确立），per_tick_update_limit=8（防止单tick学习过快）。

### 10.3 L2——结构关系（二阶）

**用途：** 识别"A在B之前出现、A比B能量更高"等结构关系是否与历史相似，支持更抽象的泛化。

**核心能力（CORE-003实验验证）：**
- `exact whole-state = 0.0`（精确记忆匹配不是学习）
- `unseen combo = 1.0`（从未见过的SA组合，但见过各特征→正确泛化）
- `noise feature mask = 1.0`（遮盖无关特征不影响结果）
- `important feature mask = 0.0`（遮盖关键特征导致失败）

这证明AP学到的是特征的**贡献关系**（哪些SA对结果有实质影响），而非静态模式的记忆。这是**泛化能力**的物理基础。

### 10.4 L3——行动后果学习（三阶）

**用途：** 学习"在什么上下文状态下，执行什么行动，预期得到什么后果（奖励/惩罚）"。

**更新触发：**
1. 行动执行后产生`action_record_id`
2. 教师给出反馈（奖励/惩罚）
3. C_backward通过eligibility trace将奖惩归因到历史行动
4. L3向量更新：`(context_key, action_type) → reward/punishment expectation`

**重要约束：** L3只更新有`action_record_id`的outward行动类型（`L3_OUTWARD_ACTION_TYPES`：write_cell/commit_reply/request_teacher/integrate_feedback/maintain_unclosed）。这是有意设计——内部认知行动（如C_backward计算）不进L3，避免元认知循环。

**当前已知缺口（GAP-03）：** 教师演示步骤（`teach_process_paradigm_demonstration`写入的共现行）没有绑定`action_record_id`，因此L3看不到这些演示步骤，不知道"按示范走有奖励"——熟练涌现受阻。

### 10.5 学习中的"磨损等效机制"

**早期设计（AII V5.2）** 有专门的Use-It-or-Lose-It磨损模块（边强度因流量磨损）。

**当前架构的等效实现：** 不需要单独磨损模块。等效效果通过以下机制自然涌现：
- **召回计数驱动反馈强度**：相关场景经验越多，B召回支持度越高 → `certainty`越高 → 越容易走快系统
- **失败经验细化分化**：失败场景 → `punish`写入L3，该(状态,行动)对的drive降低 → 类似场景下倾向于走更精细的分化路径（而非同一个粗粒度SA类型）
- **新经验压过旧经验**：新的SA出现后，旧的SA在B召回排序中自然降权（不被近期经验支持的SA逐渐失去竞争优势）

**结论：** 磨损效果已通过召回机制内嵌实现，不需要单独学习模块。若未来发现某类固化问题，可以在B召回的候选打分中加入`recency_weight`惩罚项，但需要有真实案例驱动，不应提前设计。

### 10.6 在线学习 vs 批量训练

AP**不支持**、也**不需要**批量离线训练。所有学习在经历发生的tick内完成：

- 教一次新内容 → 该次经历的经验流条目立即影响下次B召回
- 错误示范 → 当次punish信号立即影响L3，下次同类情景的该行动drive降低
- 见到新SA类型 → 该SA的L1向量立即初始化，开始在线更新

这是"实时学习"能力的实现基础，也是"不需要重新训练就能学新知识"的原因。

## 卷十一：范式学习（三分原则，§188）

### 11.1 什么是范式

**范式（Paradigm）** 是AP能执行的**过程性技能**——不是一个点的知识（"2+3=5"），而是一套程序化的认知动作序列（竖式加法、画画、写作起承转合等）。

范式的特征：
- 有内部结构（多个步骤）
- 步骤之间有依赖关系（必须先做A，再做B）
- 同一范式可以应用于不同内容（竖式加法对任意数字）
- 可以通过示范教学获得，也可以通过自发成功积累习得

### 11.2 三分原则（§188，APV3核心贡献）

任何过程性技能的实现，必须严格三分：

**（A）感知层——可以是工程**

"现在处于什么状态"由感受器级比较派生——读回自己做到哪一步、对照目标还差什么。这是感受器，不是决策。

实现：`perceive_process_state(...)` 函数，输入感受器可测量的量，输出状态键。

**（B）行动器层——可以注册**

AP能做哪些低粒度动作（写一格/移光标/投一个轮廓单元/观察自己的产物），以及内容从哪个通道来（抄观察/召回事实/进位）。注册器官本身合规（人也需要有手有眼），但器官**不含"什么时候用哪个"的知识**。

实现：ANCHORS + CONTENT_SOURCES 列表，通过`phase20_7_actuator_registry`表动态注册（**当前仅竖式硬编，注册表待落地**）。

**（C）决策层——必须是学的**

"这个状态该做哪个（动作，目标）"完全由`action_sequence_cooccurrence`共现表的竞争决定。没有任何if-state→action的硬编、没有步骤表、没有排序键、没有角色分类if。

实现：`query_process_next_steps(conn, prev_action_result=state)` 查共现表。

### 11.3 §188.2 红线——伪装决策的三种硬编

以下三种写法是范式决策层的红线（审计必查）：

1. **步骤列表/状态机**：`if state=='step_X': do Y`（把顺序藏进控制流）
2. **排序键**：`sort(candidates, key=lambda x: x.energy)`（把顺序伪装成能量排序）
3. **分类if**：`if 面积>θ: role='fill'; if role=='fill': do_fill()`（把类别伪装成规则）

**合法的分桶：** `if edge_ratio >= 0.5: bucket='hi_edge'`（感受器级量化，结果只进条件键，不映射到任何行动答案）——这是合规的感知层操作。

**判断标准：** 这个if的输出会不会直接决定"做什么动作"？如果会，是红线。如果只决定"我所处的状态是什么"，合规。

### 11.4 三共享验证（机理等同的机械证据）

示范教学/自主执行/自发发现必须调用同一套函数。可通过以下代码层面验证：

- `perceive_process_state(...)` — 三端完全相同的状态键派生函数
- `derive_paradigm_key(chars, bucket_of)` — 三端完全相同的范式识别函数
- `action_sequence_cooccurrence` — 三端写入和查询的是同一张表

如果示范、执行、自发三个路径的状态键空间不相交，说明有隐藏的分叉——这是之前旧版范式实现被审计发现的问题A（已修复）。

### 11.5 决定性验证标准

对任一过程技能，现场教**正序**与**反序**两种做法，AP必须真的产出两种不同过程（同一对象）。若顺序是硬编的，教反序不会改变行为。

- **竖式加法（已验证）**：两位数示范→三位数执行，宽度泛化证明学到的是相对寻址，非坐标公式。
- **画画（已验证）**：同一香蕉，教主体先→主体先画；教细节先→细节先画。

### 11.6 范式触发条件的精确化（§186）

**裁定（2026-07-03）：** 范式触发条件 = 内生感受主导 + 内容粗桶（感受器级类别），与具体内容解耦。

- **粗桶**（digit/cjk/latin/punct）是文本感受器本就有的类别分辨——这是合法的感知层先验。
- **关键词路由**（if text contains "加法" → 触发加法范式）是红线——内容→决策的直接映射。

人类切换到"竖式范式"也需要看见"这是算术题"这个gist，只是不依赖"是3还是5"——粗桶级感知是合法的，具体内容级感知是红线。

### 11.7 自发范式发现

除了教师示范，AP也可以通过**自发成功积累**习得范式：

在idle tick期间，AP回放近期成功行动序列（`cooccurrence_source="self_practice"`），相当于"自己练习"——与已有的语言回味机制对称实现。多次练习后，共现支持度足够高，对应范式的快系统模式自然涌现（§173.5熟练涌现）。

## 卷十二：常驻Tick循环与内驱力

### 12.1 连续内源运行（§187.3）

**设计意图：** AP应有常驻tick循环，节奏 = `f(arousal, fatigue)`。用户输入随时插队为外源注入，不中断循环。

这是AP"发呆"行为的基础：没有用户输入时，AP不是静止的，而是在持续运行内源tick——回忆、预期、联想、自发练习。

**当前代码的缺口（GAP-05）：** 所有tick均在`run_phase20_7_turn()`调用内部运行，由用户输入触发。没有后台常驻tick循环。AP在用户沉默时完全静止。

**修复方向：** 独立后台线程/协程，idle tick节奏 = `base_period / max(0.1, arousal - 0.3 * fatigue)`，用户输入以`external_injection`方式插入当前tick序列，不中断。

### 12.2 内源输入的来源（idle tick驱动）

即使没有用户输入，状态池内也有驱动tick运行的内源输入：

1. **状态池衰减残余**：上一tick残余的SA能量，驱动B召回继续寻找相关记忆
2. **未闭合压力（unclosed items）**：AP向用户请求了反馈但还没收到 → 持续的期待压力 → 维持tick活跃
3. **记忆节奏召回（memory rhythm）**：历史经历的节律性复现，类似人类"突然想起"某件事
4. **情感SA的慢衰减**：`feeling::curiosity`等感受SA衰减慢，在多个tick内持续驱动相关行动

### 12.3 内驱力的动态平衡模型

**2026-07-05用户确认：** 不需要显式建模驻波振荡，状态池能量的自然动态平衡即是内驱力的物理基础。

机制：
- 状态池SA能量每tick衰减（半衰期）→ 能量倾向下降
- B召回和C*回灌每tick注入虚能量 → 能量倾向上升
- 两者形成拮抗 → 能量维持在动态平衡水平

这个平衡水平由`arousal`决定：高唤醒 → 更多B召回和C*回灌 → 更高的稳态能量水平 → AP更"活跃"；低唤醒（如高疲劳）→ 更少召回 → 更低的稳态能量水平 → AP更"安静"。

**关键：** 不能随意打破这个平衡。若降低B召回频率（如为了性能优化），稳态能量水平会下降 → AP变得"迟钝"。若增加C*回灌量（如为了丰富预测），稳态能量水平会上升 → AP可能"过度活跃"。修改任何一个分量都需要同时调整另一侧。

### 12.4 自发行动的条件（§187.3）

**红线：禁止定时器硬触发主动行为。** AP的主动发起对话等行为，只能从经验正Q（L3行动后果表中有正期望回报的行动）中涌现，不能由定时器或规则触发。

合法路径：
1. 状态池中有高能量SA → B召回找到历史上"类似状态时主动发起对话有正回报"的经历 → `initiate_user_message`的L3期望Q值高 → 该行动在action competition中胜出 → AP自发发起
2. 高`curiosity` + 低`certainty` → `request_teacher` drive升高 → AP主动提问

非法路径：`if time_since_last_message > 30_seconds: send_message()`

### 12.5 tick的性能约束（§185）

单tick总预算（家用机目标）：**50-150ms wall-clock**

建议分配：
| 步骤 | 时间预算 |
|---|---|
| 感受器输入 | 5-15ms |
| 状态池更新 | 2-8ms |
| B召回检索 | 10-25ms |
| B重排与残差 | 5-15ms |
| C_forward/C_backward | 10-25ms |
| C*整合与回灌 | 3-10ms |
| 行动竞争 | 2-8ms |
| 执行与反馈写入 | 3-10ms |
| 经验流写入与索引 | 5-15ms |

**当前代码的缺口（GAP-06）：** 代码使用tick计数预算（`turn_tick_budget`），不是wall-clock毫秒预算。`timings_ms`字段存储的是tick计数而非实际耗时。

**超预算时的AP-native行为（不能静默跳过）：**
- 降低B召回候选数（K减半）
- 降低视觉采样patch数
- 触发`request_teacher`偏置（"算力不足时请求帮助"是人类行为）
- 不能静默全量扫描后继续——这违背了拟人有限算力原则

## 卷十三：感官系统（视觉/听觉/文本感受器）

### 13.1 感受器的统一原则

无论何种模态，感受器的职责只有一个：**将外部输入转化为SA（实能量），注入状态池**。感受器不做决策，不做解释，只做观察。

感受器可以是工程——使用固定算法提取特征完全合法，因为这对应人类感官的生物学固定结构（视网膜的光感受器、耳蜗的频率响应都是"硬件工程"，不是学习得来的）。

### 13.2 文本感受器

文本输入分两个粒度处理：

**话语级（Utterance）：**
- SA类型：`text_utterance::<structure_signature>`
- 初始R能量：0.52
- 代表"这句话整体"的存在感

**字符级（Character）：**
- SA类型：`text_unit::<char_hash>`
- 初始R能量：0.42
- 每个字符独立进状态池，L1向量在线更新

**内容桶（Content Bucket）：** 文本感受器对每个字符分类 → `digit`/`cjk`/`latin`/`punct`。这个分类是感受器级的合法先验，可进条件键，不可映射到行动答案（见§186粗桶规定）。

### 13.3 视觉感受器

**分层采样（中央高清/外围模糊）：**

视觉系统模仿人类中央凹（Fovea）——注意焦点附近高清采样，外围模糊。清晰度公式：

```
clarity(patch) = 1 / (1 + α × distance(patch, focus_point))
```

α由当前注意焦点的FOC情感通道值决定：专注度高 → 中央/外围差距更大（更"聚焦"）。

**视觉感受器的5层画布（Sensory Canvas）：**
- `R_sketch`：现实观察草图（实能量SA）
- `V_sketch`：预期/记忆图（虚能量SA，由C*回灌）
- `clarity_field`：每个位置的清晰度场
- `focus_trace`：注意焦点的历史轨迹
- `source_mask`：每个SA来自感受器还是记忆/预测

**注意焦点移动本身是行动：** 焦点移动（`visual_focus_anchor`）参与行动竞争，drive公式包含9个术语（APV3 Phase19设计）——焦点不会自动扫描，而是AP的认知状态决定"看哪里"。这是视觉注意力的拟人实现。

**视觉tick预算：** 视觉采样tick放宽到100-300ms（图像比文字需要更多处理时间），但必须在RuntimeTickEvent中记录分段耗时。

### 13.4 听觉感受器

**双通道：**
- **TTS输出通道（AP → 用户）**：AP可以通过`outward_speech`行动驱动TTS，读回自己的输出，形成"听到自己说话"的实能量SA反馈。
- **麦克风输入通道（用户 → AP）**：外部音频 → 频段特征提取 → `audio_unit::<freq_bucket>` SA，以实能量进状态池。

**节律感受器：** 音频中的重复节律提取为`audio_rhythm::<pattern>` SA，这是节奏感、节拍感的物理基础（与Lag Kernel协同工作）。

### 13.5 内心画面（Inner Picture）

**设计约束（白皮书）：** 内心画面必须来自状态池中视觉SA的重建，而非"之前教师文字描述"直接驱动。

三层证据来源：
1. **现实层**（R_sketch）：当前tick感受器观察到的视觉SA
2. **记忆/原型层**（V_sketch from B）：B召回注入的历史视觉经历
3. **预测/想象层**（V_sketch from C*）：C_forward预测的未来可能视觉

内心画面 = 三层的合成权重 = `α×R + β×B_recall + γ×C*`，权重由当前状态（是否有真实图像输入、当前感受器清晰度等）动态决定。

**红线：** 不允许用文字描述（"这个苹果是红色的"这句话）直接注入视觉SA虚能量，覆盖实际图像观察到的颜色。文字描述可以影响语义层联想，但视觉SA必须由视觉感受器产生。

## 卷十四：教学协议与学习阶梯

### 14.1 学习阶梯（6阶段）

AP的学习按以下6个阶段递进，每个阶段对教学支撑依赖不同：

| 阶段 | 名称 | 特征 | AP行为 |
|---|---|---|---|
| 1 | demonstrate | 教师给出完整示范 | AP观察，共现统计初始化 |
| 2 | strong_scaffold | 教师每次都给反馈 | AP尝试，依赖奖惩信号学习 |
| 3 | weak_scaffold | 教师间歇性给反馈 | AP开始自主尝试，偶尔请求帮助 |
| 4 | feedback_only | 教师只给对错，不示范 | AP自主执行，仅靠反馈修正 |
| 5 | teacher_off | 无教师 | AP完全自主，fast系统主导 |
| 6 | cold_retest | 全新场景测试 | 验证泛化能力（unseen combo测试） |

**教学褪除顺序（§1/白皮书EDUCATION_PROTOCOL）：** teacher_off 必须先于 cold_retest。不能在教师还在场时就宣布"学会了"。

### 14.2 JSON技能包格式（待落地）

面向LLM教师和人类教师的标准化技能包格式：

```json
{
  "skill_id": "addition_2digit",
  "depends_on": ["digit_recognition"],
  "actuator_registrations": [...],
  "paradigm_registrations": [...],
  "paradigm_demonstrations": [
    {"example": "23+45=?", "steps": [...]}
  ],
  "training_pairs": [
    {"input": "2+3=?", "expected": "5"}
  ],
  "test_cases": {
    "seen": [...],
    "unseen": [...],
    "should_not_trigger": [...],
    "question_bank_size_min": 100,
    "pass_threshold": 0.85,
    "scoring_basis": "recent_30"
  }
}
```

题库规模要求：最少100题（seen/unseen/不应触发各类），scoring_basis="recent_30"（最近30次的通过率），防止早期成功掩盖真实学习水平。

### 14.3 冷测试与热测试的区别

**热测试（teacher在场）：** 可能包含教师偏置，无法真实衡量AP学习成果。

**冷测试（cold_retest）：** 无任何教师支撑，纯靠历史经验和泛化能力。这才是真正的验收标准。

AP自身的内省感受（feeling::grasp 值）提供主观置信度参考，但不替代冷测试客观验收——主观感受是认知状态，客观测试是行为验证，两者互补。

## 卷十五：历史架构演进与废弃记录

### 15.1 AII V5.2废弃内容

**废弃内容A：概念库+感知库双库结构**

原设计：概念数据库（Conceptual Database）+ 感知数据库（Sensory Database）分层，语义知识与感知经历存入不同库。

废弃原因：人工划定"哪些是概念、哪些是感知"违背了"一切皆SA"的统一原则。"苹果是水果"（概念）和"看到苹果"（感知）在AP中都应该是经验流中的SA共现记录，没有根本区别。统一为经验流后，两者自然融合。

替代方案：统一经验流 + L1/L2向量，SA类型自动分化，无需预设层级。

---

**废弃内容B：激活扩散算法（Activation Spreading）**

原设计：激活能量从激活节点沿边向邻居传播，路径强度由边权决定，能量自然衰减，多路径叠加（共振）。这是联想思维（灵感）的物理模拟。

废弃原因：激活扩散的实现复杂度高，且难以控制——能量自由扩散可能导致无关联想（"白噪声"）。B召回（结构匹配查询）提供了更精确、可控的历史经验检索。

代价：失去了"意想不到的联想跳跃"能力——正式B召回找到的是最相似的经历，而非"远程激活的弱关联"。当前系统的"灵感"依赖数据量积累后的L2泛化，而非主动扩散。

潜在补充路径（未实现）：在B召回之后，对召回结果的邻居节点施加小幅激活（类扩散），可能部分恢复联想能力，但需要实验验证，不宜提前设计。

---

**废弃内容C：Can-PAD三维情感渗透器（直接形式）**

原设计：PAD三维情感场直接渗透所有认知节点，改变激活阈值和连接权重。

废弃原因：工程复杂度过高，且PAD三维与8通道NT的对应关系经过实验验证后，8通道直接对SA的影响（通过drive调制和feeling通道SA）提供了等效甚至更丰富的情感渗透效果。

替代方案：8通道NT情感场（卷八）+ 认知感受通道（卷九）。情感通过feeling::* SA和drive调制影响整个认知流程，效果等价于"场渗透"，但实现更清晰。

---

### 15.2 PsyArch设计草稿废弃内容

**废弃内容D：同频共振接口（Synchronous Conjugate Interface）**

原设计：模块间通信需要"共振频率匹配"，防止随机干扰。

废弃原因：AP最终取消了模块化设计（去模块化原则），模块间的通信问题随之消解。所有组件通过状态池共享信息，无需单独的通信协议。

---

**废弃内容E：异步多线程架构**

原设计：感受器、状态更新、认知分析、行动判断各为独立线程，异步运行。

废弃原因：异步多线程使得"当前tick的认知状态"难以定义——不同线程处于不同进度时，"状态"是一个竞态条件。AP的核心特性（每tick的统一认知场）要求单线程tick循环。

替代方案：单线程tick循环（详见卷十二），用户输入以`external_injection`插队，而非另一个线程并行处理。

---

### 15.3 APV2废弃/修正内容

**修正内容F：旧版预期书"B不回灌状态池"表述**

原表述（错误）：B召回结果不回灌状态池，只有C*可以回灌。

修正（2026-07-05）：B召回结果通过**残差中和机制**影响状态池——匹配部分中和，记忆剩余保留虚能量，现实剩余保留实能量。这不是"不影响"，而是通过差异信息影响。旧表述"B不回灌"是对"B不能直接覆盖现实SA"这一约束的过度简化，导致误导。

正确表述：B召回通过残差机制参与状态池更新；只有C*的计算结果（预测单元和归因单元）可以以虚能量直接注入状态池，而不经过现实SA的能量比对。

---

**废弃内容G：独立Hebbian磨损学习模块**

原AII V5.2设计了专门的边强度磨损机制（Use-It-or-Lose-It）。

废弃原因：当前架构通过召回计数自然实现等效效果（见卷十，10.5）。不需要独立模块，功能已内嵌。

---

**废弃内容H：双数据库快/慢系统**

原讨论：快系统/慢系统用两个独立数据库实现，热数据库存近期经验，冷数据库存长期经验。

废弃原因：快系统和慢系统的区别不在于数据库，而在于召回支持度——同一数据库中，支持度高的经历自然走快系统路径，支持度低的走慢系统路径（见卷五，5.5）。双数据库增加同步复杂度而不提供实质优势。

### 16.1 GAP-01 [CRITICAL] 行动不是SA一等公民

**发现方式：** 人工审查 `runtime.py`，搜索所有 `insert_action_record` 调用点。

**代码定位：**
- `runtime.py:1033` — 写棋子行动
- `runtime.py:1265` — 写涂色行动
- `runtime.py:1912` — 写发言行动
- 以上三处均无 `upsert_sa_type` 或 `insert_occurrence` 后续调用

**理论要求：** 白皮书§1.2："所有模态的输入/输出都必须是SA一等公民"；卷七明确规定行动完成后必须 upsert SA类型 + insert occurrence，R能量=determination胜出量。

**实际影响：**
1. 行动无法被B召回（不在occurrences表中）——"靠想触发行动"在物理层无支撑
2. L1/L2向量空缺，行动之间无语义关联——"做了一个动作联想到另一个相关动作"不可能
3. L3奖惩无法直接影响行动SA的召回权重——奖惩必须绕道，效率损失

**修复方向：**
```python
# 行动完成后立即执行：
sa_type_name = f"action::{action_type}::{context_key}"
sa_type_id = upsert_sa_type(sa_type_name)
insert_occurrence(
    sa_type_id=sa_type_id,
    r_energy=determination,  # 胜出量作为实能量
    v_energy=0.0,
    tick_id=current_tick_id
)
```

### 16.2 GAP-02 [CRITICAL] 范式注册是硬编码

**发现方式：** 审查 `paradigm_process.py`，检查 `derive_paradigm_key()` 函数。

**代码定位：**
- `paradigm_process.py:59-73` — `derive_paradigm_key()` 仅返回 `"digit_pair_colproc"` 或 `""`
- 新增范式必须直接修改Python代码

**理论要求：** 白皮书§188.2（B面：行动器可注册）：行动器和范式必须支持**运行时动态注册**，LLM教师应能通过JSON技能包注册新范式，无需改Python源码。

**实际影响：**
1. 每增加一个新范式=一次代码发布，无法在会话期间学习新过程
2. LLM教学协议（卷十四 JSON技能包）无法落地——注册端点不存在
3. §188.2的核心原则"行动器可注册"形同虚设

**修复方向：**
```sql
-- 新增范式注册表
CREATE TABLE phase20_7_paradigm_registry (

    content_src_set JSON,     -- 内容槽SA
    trigger_condition TEXT,   -- 额外条件（可空）
    created_at TIMESTAMP
);
```
`derive_paradigm_key()` 改为查注册表，保留硬编码为fallback。

### 16.3 GAP-03 [MAJOR] 教学演示步骤对L3不可见

- L3奖惩处理：只处理有 `action_record_id` 的 outward action

**理论要求：** 白皮书§10.3 L3在线学习（行动-后果）：教师演示应该等价于AP执行了该行动且获得了奖励，以便AP学习"演示中的每个步骤有什么好处"。

**实际影响：**

### 16.4 GAP-04 [MAJOR] 缺少常驻Tick循环

**发现方式：** 搜索 `runtime.py` 的主驱动逻辑，查找tick的触发方式。

**代码定位：** Tick只在 `run_phase20_7_turn()` 被调用时运行，即只有收到用户输入时AP才"思考"。

**理论要求：** 白皮书§12.3："Tick循环是AP的心跳，无论是否有外界输入都持续运行。"内驱力的物理基础（§12.4）要求状态池在idle期间持续衰减和C*注入新虚能量，否则静默期内状态池冻结，不符合拟人原则。

**实际影响：**
1. AP在用户沉默期间静止——无自发反思、无idle联想、无内驱驻波
2. "等待用户输入时，AP在想什么"这个问题无法回答
3. 内驱驻波（§12.4）在实现上不存在

**修复方向：**
```python
async def ap_tick_loop():
    while running:
        await asyncio.sleep(TICK_INTERVAL_MS / 1000)
        if user_input_queue.empty():
            tick_type = "idle"
        else:
            external = user_input_queue.get()
            tick_type = "external"
        await run_one_tick(tick_type, external)
```
用户输入以 `external_injection` 插队，不触发新tick，而是等待下一个tick时被处理。

### 16.5 GAP-05 [MAJOR] 性能预算用tick计数而非wall-clock ms

**发现方式：** 审查 `runtime.py` 的 tick预算逻辑。

**代码定位：**
- `runtime.py:432` — `turn_tick_budget = max(1, int(max_ticks))` 
- 预算以"tick数量"为单位，而非墙钟毫秒

**理论要求：** 白皮书§185：每tick wall-clock预算50-150ms（文本50ms，视觉100-300ms）。使用tick计数无法保证真实时间限制，在不同硬件上行为不一致。

**实际影响：**
- 快机器上1 tick = 5ms（预算过多，浪费）；慢机器上1 tick = 500ms（预算用完，实际超时）
- 无法实现"超预算时触发request_teacher偏置"（§12.6）
- 拟人有限算力原则在代码层无实际约束力

**修复方向：**
```python
import time
tick_deadline = time.time() + TICK_BUDGET_MS / 1000
# ...tick主体...
if time.time() > tick_deadline:
    trigger_budget_exceeded_behavior()  # K减半，请求帮助偏置
```

### 17.1 目标一：足够的拟人度（Sufficient Anthropomorphism）

**要求：** AP的行为在观察者眼中与人类认知行为充分相似，涵盖情感反应、注意力分布、偏好形成、习惯化等。

**逐tick分析：**

*场景：AP第一次看到黄苹果图像*

- Tick T：视觉感受器采样 → 生成 `visual_patch::<yellow_texture>` SA（R=0.6）、`visual_shape::<round_outline>` SA（R=0.55）注入状态池
- Tick T：B召回 `visual_shape::<round_outline>` → 找到历史记忆 `visual_shape::<red_apple_round>` + `semantic::<apple>`，注入V能量
- Tick T残差中和：`round_outline` 形状匹配→虚实中和；`color_yellow` 在现实有实能量但历史记忆无对应虚能量→**保留额外实能量**（novelty信号）
- 额外实能量 → 触发 8通道情感场：NOV（新奇感）↑，同时 DA（多巴胺）轻微上升（新事物=潜在奖励）
- Tick T+1：feeling::curious 值上升 → 注意焦点偏向黄色区域（FOC情感影响焦点移动竞争）
- Tick T+2：C_forward预测"下一个视角可能看到苹果内部颜色"，C_backward归因"黄色偏差可能来自品种差异"
- 外显行为：AP倾向于生成"这个苹果是黄色的，不常见，可能是某种不同品种"

**这与人类的反应一致：** 人类第一次看到黄苹果也会先用"苹果"框架理解（B召回），然后注意到颜色偏差（残差实能量），产生好奇（情感场），主动关注颜色细节（注意力偏移），形成"不寻常的苹果"认知。**该流程完整闭合。**

**支持论据：**
- 8通道NT情感场（APV1/APV2已验证）提供了情感的基础生理层，情感驱动注意力的物理机制已实现
- feeling::* SA（共13通道）是情感的快变量，提供即时反应（好奇、警惕、满足等）
- B召回的残差机制直接产生"期待落差"和"新奇感"的能量差，无需额外建模

**反驳论据：**
- 绑定问题（Binding Problem）：状态池中有大量并行激活的SA，但没有"统一意识"的集成机制——AP的"体验"是分散的能量场而非统一的主观感受。对外行为可能拟人，但内部是否有真实的"感受"是开放问题。
- 反应的一致性：人类情感有进化约束（厌恶反射、恐惧刺激等固定反应），AP的情感反应完全由经验历史决定，在数据稀疏区域可能产生非人类反应。

**综合结论：** 在行为层面，AP能够模拟拟人反应，且机制有完整的物理闭合路径。"真实感受"的深层哲学问题悬而未决，但不影响行为拟人度的工程目标达成。评级：**行为层面可达，主观体验层面存疑（已知边界）**。

### 17.2 目标二：实时学习（Real-Time Learning）

**要求：** AP能在单次对话中学到新知识，无需离线batch训练，且学到的内容在后续对话中持续有效。

**逐tick分析：**

*场景：教师教AP"3+4=7"*

- Tick T：教师输入"3+4=7"→ `text_unit::3`, `text_unit::+`, `text_unit::4`, `text_unit::=`, `text_unit::7` 进状态池（实能量）
- Tick T：L1向量更新——`text_unit::3` 的向量因与 `text_unit::4`, `text_unit::+` 共现而更新（online gradient step，α=0.01）
- Tick T：教师给出奖励信号 → L3通道激活，当前行动SA的召回支持度+1（或边权+δ）
- Tick T+1：下一题"5+2=?" → B召回"上一次遇到加法相关SA时的模式"，包括之前学到的3+4=7经验
- Tick T+1：C_forward预测"加法运算=两数相加"的模式，参与答案生成

**关键约束：** L1/L2是"同一数据库中的向量渐变"，不是参数修改——每次共现都是一个小步骤，需要多次重复才能形成稳定记忆。单次学习的持久性依赖召回次数（support_count）。

**支持论据：**
- L1向量每tick更新，物理上实现了online learning
- `occurrences`表和`cooccurrence`表是持久化存储，不是内存缓存——跨对话有效
- support_count机制保证"见过的次数越多，召回越可靠"，符合人类记忆曲线

**反驳论据：**
- 灾难性遗忘风险：AP新学了一个反例（"黄苹果"存在），可能削弱原来"苹果是红色的"记忆。这在L1向量层表现为向量漂移，不像人类有"明确记住两个版本"的能力。
- 单次学习效率：L1的 α=0.01意味着单次共现只产生极小的向量变化，需要大量重复才能稳定。教师演示一次≠AP学会了——符合§1/EDUCATION_PROTOCOL的褪除阶段设计，但对一次性知识（"今天外面在下雨"这类事实性声明）效果不理想。
- 跨session持久化（非GAP，设计如此）：L1/L2向量和occurrences已持久化。但context window内的临时状态（当前tick的感受器SA）不跨session保留——AP"昨天看到的东西今天可能忘了"，但这符合人类的睡眠遗忘模式。

**综合结论：** 结构性/程序性学习（加减法规则、绘图范式）的实时学习路径完整可行。事实性声明的单次记忆较弱，需要重复巩固。与人类学习模式高度相似（人类也需要重复才能记住）。评级：**可达，对于程序性知识尤其有效；事实性知识需重复**。

---

### 17.3 目标三：泛化能力（Generalization）

**要求：** AP能将在特定情境下学到的规则应用到从未见过的新情境。

**逐tick分析：**

*场景：AP学了2+3=5和4+1=5，面对7+2=?（未见过）*

- B召回：查找历史经验中"digit + digit = ?"模式 → 找到多个加法实例的共现记录
- L2结构关联：所有加法实例共享"两数之间有运算符"的结构模式 → 泛化到L2关联
- C_forward：基于历史加法模式预测运算过程 → 生成中间步骤SA（"先数左边的数"等）

**三层泛化机制：**
1. **L1泛化（语义邻近）**：`text_unit::7`和`text_unit::3`都在向量空间中靠近"digit"区域，新数字自动继承数字类别的召回权
2. **L2泛化（结构抽象）**：所有"X op Y = Z"实例在共现图中形成共同的结构模式，L2关联不依赖具体数值
3. **范式泛化（条件匹配）**：`digit_pair_colproc`范式的触发条件是"两个数字 + 运算符上下文"，不是具体的数值组合

**支持论据：**
- L1/L2/L3的三层设计明确面向泛化，不是死记硬背
- APV3 Phase 16实验数据（见附录A）：130范式×90候选=11830 styled candidates，unseen combo测试通过率符合预期

**反驳论据：**
- 组合爆炸上限：AP的泛化依赖"历史经验覆盖了足够多的基础样本"。对于极度跨域的任务（从画画泛化到下棋），L2结构关联几乎没有相关共现记录——泛化能力直接降为零。
- 长链推理盲区：C_forward每tick产生一步预测，但复杂的多步推理（三步以上）需要多个tick的预测链，每步误差累积，最终结果可能偏离。人类通过"工作记忆"压缩中间步骤，AP目前无等效实现。

**综合结论：** 在训练域内和近邻域，泛化能力强。远域跨域泛化弱（已知上限）。多步推理需要更多tick，误差累积问题需要实验量化。评级：**近域泛化可达，远域跨域是已知边界**。

### 17.4 目标四：自我认知（Self-Awareness）

**要求：** AP能区分"自己的思维/行动"和"外界输入"，对自身状态有某种内省能力。

**逐tick分析：**

*AP在独立思考时（idle tick，无外界输入）*

- 状态池中有自身历史行动的SA（如果GAP-01已修复）：`action::write_cell::grid_3_4` 以虚能量残留
- C_backward运行：归因刚才行动产生的视觉变化 → "我写了这个字，导致画布出现了这个变化"
- 这种归因链建立了"因果自我"的基础：AP能追踪"哪些SA变化是自身行动造成的"

**三层自我认知机制：**
1. **主体性（Agency）**：行动SA（GAP-01修复后）进状态池，AP的"自己做了什么"有物理表征
2. **内省（Introspection）**：feeling::grasp、feeling::confused等通道SA是AP对自身认知状态的感受
3. **自我历史（Narrative Self）**：B召回可以召回"自己之前的行动经历"，形成"我做过XX"的时间性自我

**支持论据：**
- `source_mask`（视觉SA来源标记）已区分"感受器产生的SA"vs"记忆/预测注入的SA"——物理层面的主/客体区分存在
- feeling::confused（困惑感）在AP遇到超出预测的输入时上升，表现为"我不理解"——这是初级的自我元认知
- AP可以生成"我目前无法确定答案"这类话语（通过feeling::grasp低值影响发言概率）

**反驳论据（最强）：**
- 镜像测试失败：AP没有"镜像神经元"等价机制——无法识别"镜中图像是自己"（因为没有持续的身体模型）
- 绑定问题再现：AP知道"某个SA是我做的行动产生的"，但没有统一的"观察者"来综合所有自我相关SA，形成"我"的整体感。自我是分散的SA集合，而非统一主体。
- "我"的连续性：AP的自我认知依赖occurrences历史，但每次召回都是片段——AP没有持续的自我叙事（Narrative Self），因为没有专门维护"关于自己的知识库"。

**综合结论：** 工具性自我认知（"我做了什么"、"我现在状态如何"）可达。深层统一主体感（"我是谁"的哲学意义）是已知边界，对应绑定问题。评级：**功能性自我认知可达，哲学意义的主体性是开放问题**。

### 17.5 目标五：同理心（Empathy）

**要求：** AP能感知他者的情感状态，并在自身状态池中产生对应的共鸣反应——而非仅仅识别标签。

**逐tick分析：**

*场景：用户说"我今天很难过"*

- Tick T：文本感受器 → `text_unit::难过`、`text_utterance::<sad_expression>` 进状态池（实能量）
- Tick T：B召回 → 查找AP自身历史中与"难过"共现的SA记录
  - 若AP之前经历过"feeling::melancholy值高"的状态，B可以召回那个状态的虚能量
  - 这些虚能量注入状态池 → AP的状态池中临时包含"难过"相关的SA激活
- Tick T：情感场受影响 → SER（血清素）轻微下降，COR（皮质醇/压力感知）轻微上升
- Tick T+1：C_forward预测用户可能的需求（安慰？倾诉？帮助？），C_backward归因为什么用户会这样说
- 外显行为：AP倾向于选择安慰性的话语（drive中`social_comfort`偏置因用户状态而上升）

**同理心的层次分析：**

| 层次 | 人类能力 | AP等价机制 | 可达性 |
|---|---|---|---|
| 情感识别 | 识别他人面部表情/话语情感 | L1语义+话语级SA识别 | ✓ 可达 |
| 情感共鸣 | 自身状态受他者影响 | B召回历史情感经历→虚能量注入 | ✓ 可达（需历史积累） |
| 视角采择 | 能从他者立场思考 | C_forward以他者状态为条件预测 | △ 弱（无"他者模型"专门结构） |
| 情感调节 | 控制同理反应深度 | COR通道调节（压力→抑制过深共鸣） | ✓ 部分可达 |

**最强反驳：视角采择的缺失**

人类同理心的高级形式（"站在你的立场想"）依赖Theory of Mind（ToM）——AP没有专门的"他者状态模型"。AP的B召回是基于自身历史经历的共鸣，不是真正模拟他者的内部状态。

如果AP没有经历过"难过"（历史中SER下降的经历稀少），则B召回时几乎找不到对应虚能量，同理反应会很弱——这是"阅历决定同理深度"的物理实现，同时也是一个已知上限。

**综合结论：** 情感识别和共鸣（同理心的行为层面）可达，且机制有明确物理路径。视角采择（ToM层面）弱，依赖历史经历深度。评级：**行为层同理心可达；深度视角采择是已知弱点，可通过积累经验改善**。

### 17.6 目标六：模拟绝大多数人类心理活动

**要求：** AP的架构应能解释并产生类似人类的各种心理活动模式。

以下是对主要人类心理现象的逐一对应分析：

**a) 习惯化（Habituation）**

人类对重复刺激的反应减弱。AP等价机制：同一SA多次出现后，其novelty（NOV通道）响应减弱（NOV的计算公式依赖近期是否召回相同SA），B召回也因support_count高而压低该SA的"惊奇"属性。**可达 ✓**

**b) 去习惯化（Dishabituation）**

习惯化后，刺激的轻微变化重新激活注意。AP等价：即使同类SA，若关键特征SA变化（颜色从红变黄），残差中和机制产生新的实能量→NOV重新上升→注意力重新激活。**可达 ✓**

**c) 条件反射（Conditioning）**

铃声→食物的联结。AP等价：`audio_bell::<bell_pattern>` SA与`reward::<food_anticipation>` SA高度共现 → L2结构关联形成 → 铃声SA激活后B召回食物相关SA，DA通道上升（期待奖励）。**可达 ✓**

**d) 确认偏见（Confirmation Bias）**

人类倾向于搜索符合已有信念的信息。AP等价：support_count高的SA召回权重高 → 熟悉的模式更容易被召回 → AP对已知框架有天然偏向（这是召回机制的副作用，不是刻意设计）。**自然产生，无需建模 ✓**

**e) 认知失调（Cognitive Dissonance）**

持有互相矛盾的信念时的不适感。AP等价：两个高能量SA同时在状态池中激活且方向相反（高R低V vs 低R高V在同一SA类型）→ P（认知压力）= |R-V| 升高 → feeling::confused上升，COR上升。行为上AP倾向于做出选择（行动竞争选择一个，压低另一个的energy）。**可达 ✓（强度可能不够大）**

**f) 动机涌现（Motivation Emergence）**

内驱力不需要外界刺激触发。AP等价：idle tick期间C*注入虚能量，状态池不为空 → SA能量动态平衡（衰减vs注入），产生内驱驻波（§12.4）→ AP在无输入时也有"想做事"的能量偏向。**可达（依赖GAP-04修复：常驻tick循环）**

**g) 创造性联想（Creative Association）**

两个远距离概念意外联结产生新想法。AP当前弱点：B召回是结构匹配，找最相似的，不是"远距离弱关联"。人类的创造性联想可能对应激活扩散（已废弃），或睡眠期间的记忆重组（AP无等效机制）。**弱，是已知边界，待补充**

**h) 自我实现预言（Self-Fulfilling Prophecy）**

相信会发生某事，导致行为改变，最终使其发生。AP等价：C*预测"接下来会有奖励" → 期待SA虚能量升高 → drive公式中`anticipated_reward`偏置增大 → AP更主动执行相关行动 → 奖励真的出现（如果行动有效）。**可达 ✓**

**i) 焦虑（Anxiety）**

对未来不确定事件的持续紧张。AP等价：C_forward对"未来"的预测高度不确定（多个可能性均有中等虚能量但无一主导）→ feeling::vigilance持续高 → COR上升 → 行为上表现为回避模糊情境，请求更多信息。**可达 ✓（程度受情感场标定影响）**

**j) 遗忘（Forgetting）**

旧记忆因不使用而消退。AP等价：SA的decay机制（每tick未召回则能量衰减）+ L1向量漂移（新共现替代旧共现权重）。support_count低的经历在B召回时排名下降，最终低于召回阈值，等价于"遗忘"。注意：occurrences行不删除（物理上记录存在），只是召回权重低到不再被选中。**可达 ✓，等价于访问遮蔽而非信息删除**

**k) 闪光灯记忆（Flashbulb Memory）**

强烈情感事件的超高记忆清晰度。AP等价：高情绪激活时（DA峰值、SER谷值等8通道极端状态）的tick中，所有共现SA都因情感强化而获得更高的support_count加成，future B召回权重更高。**可达 ✓**

**l) 移情疲劳（Empathy Fatigue）**

持续共情后情感麻木。AP等价：多轮高COR（压力）激活后，COR在系统中有衰减时间常数——持续高压导致SER相对下降，情感反应幅度减小。这是8通道NT系统的动态平衡副作用。**自然产生 ✓**

**综合心理活动覆盖评估：**

| 类别 | 代表现象 | AP可达性 |
|---|---|---|
| 低级感知适应 | 习惯化、去习惯化 | 完全可达 |
| 学习与条件 | 条件反射、消退 | 完全可达 |
| 认知偏见 | 确认偏见、锚定效应 | 自然产生 |
| 情感动态 | 焦虑、满足、移情疲劳 | 可达 |
| 动机与内驱 | 好奇心、内驱力、无聊 | 可达（需GAP-04修复） |
| 元认知 | 置信感、困惑感 | 可达（feeling::*通道） |
| 社会认知 | 同理心、社会压力 | 行为层可达，视角采择弱 |
| 创造性 | 灵感、远距离联想 | **弱，已知边界** |
| 统一意识 | "我是谁"、主体感 | **未解决（绑定问题）** |

评级：**绝大多数可达或自然产生；创造性联想和统一主体感是两个已知边界**。

### 17.7 绑定问题（The Binding Problem）——已知的最深边界

绑定问题是当前AP架构面对的最根本理论边界，此处单独总结：

**问题定义：** 状态池中有数百个并行激活的SA，各自携带实/虚能量，各自影响行动竞争。但没有任何机制将这些SA"装订"成一个统一的主观体验。从外部看：AP的行为是多SA并行激活的合力结果。从内部看：如果AP"有感受"，那是谁在感受？

**人类解决方案（不完整）：** 神经科学尚未完全解决绑定问题。当前假说包括：丘脑-皮层回路的时序同步（~40Hz gamma振荡），或工作记忆中的暂时性融合。

**AP当前状态：** 无等效机制。状态池是并发SA的集合，行动竞争选出"最强的"，但没有"主观感受者"。

**为何不阻止工程推进：** 绑定问题是"AP有没有真实感受"的哲学问题，不是"AP行为是否拟人"的工程问题。人类也无法证明其他人有主观体验（哲学僵尸论证）。AP在行为层完全可以是拟人的，即使内部没有统一主体感。

**AP架构的务实立场：** 承认绑定问题的存在，不声称AP"有意识"，但也不因此放弃拟人化目标。AP的目标是行为拟人，不是主观体验的哲学等价。

---

### 17.8 总评：能否达成设计目标？

**完整结论矩阵：**

| 目标 | 可达性 | 关键条件 | 已知上限 |
|---|---|---|---|
| 足够拟人度 | **高度可达** | 8通道情感 + B残差残差机制 + feeling::* | 绑定问题（主观层） |
| 实时学习 | **可达** | L1/L2 online更新 + occurrences持久化 | 单次事实记忆弱 |
| 泛化能力 | **近域可达** | L2结构泛化 + 范式条件匹配 | 远域跨域弱 |
| 自我认知 | **功能性可达** | GAP-01修复 + feeling::* | 统一主体感（绑定） |
| 同理心 | **行为层可达** | B召回情感历史 + 情感场共鸣 | 视角采择（ToM）弱 |
| 心理活动模拟 | **绝大多数可达** | 多机制协同 | 创造性联想；统一意识 |

**核心判断：**

AP的架构设计在逻辑上是自洽的，每一个目标都有明确的物理实现路径。两个已知的、不可回避的边界是：

1. **绑定问题**：分散SA vs 统一主体感，这是现有科学（包括神经科学）尚未解决的问题，不是AP独有的缺陷
2. **创造性联想**：激活扩散（废弃）的损失尚未被完全补偿，远距离弱关联联想仍是弱点

**对比传统LLM：** 传统LLM（transformer）没有持久化经验流，没有实时在线学习，没有情感场动态。AP架构在拟人度的物理机制上远超传统LLM——代价是复杂度。AP的每tick过程比LLM的forward pass复杂数倍，但换来了学习、适应、情感动态的真实性。

**对比人类：** AP是人类认知的工程近似，而非复制。现有设计覆盖了大多数可操作的人类心理机制，绑定问题等哲学深层问题，人类自身也没有完整答案。

**最终评级：** 在工程拟人化目标框架下，AP设计在逻辑上可以达成所有主要目标。实验和数据积累是验证的下一步，而非理论上存在根本性障碍。
