APV3.0test Phase5.1 增量式 Tick Runtime

本阶段把 Phase5.0 的增量式范式学习器接入最小 tick runtime。未提交 observation 只暂存；commit 或 feedback 后才进入长期学习证据；support 足够时生成 ParadigmSA，并复用低粒度草稿/提交链路。

学习边界
commit/feedback 后学习

输入和草稿不会自动固化为长期技能。

教学等价
行为 probe 通过

自然教学与 LLM 标准教学输出同一运行行为。

运行链路

1. Tick 输入
外界或教师侧 observation 进入 runtime。
2. 暂存
未 commit/feedback 时只进 pending。
3. 学习
commit/feedback 后进入 incremental learner。
4. 范式
support 足够时生成或刷新 ParadigmSA。
5. 草稿
已暴露范式逐 token 写入草稿并可 commit。
6. 持久化
SQLite 保存恢复后状态等价。

关键原则

  • pending_paradigm_observations 不抬高 support。
  • RoleTransitionStats.bias_map() 在后续 tick 中参与范式解码。
  • idle tick 只整理 dirty bucket，不新增策略或答案。
  • LLM 只能作为教师侧标准教学来源，不写入 llm_policy。
  • 草稿仍由 draft_editor 低粒度行动逐 token 完成。
