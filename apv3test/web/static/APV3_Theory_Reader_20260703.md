## 0. 一句话回答 "AP 是什么"

**AP（Artificial PsyArch，人工心智架构）是一种白箱、持续、全栈拟人的认知架构。它以 tick 为周期运行，每一 tick 走完整闭环：感受 → 状态池 → 短期结构池 → 召回 → 预测/追溯 → 认知感受 → 情绪 → 行动竞争 → 反馈 → 学习。**

它**不是** transformer，不是单次"看 context 吐下一个 token"；它**不是**符号主义答题机；也**不是**"拿 LLM 包一层 UI"的 chatbot。

它是一个真实运转的认知底座：你教它，把教到的东西写进**经验流**；它下一轮怎么说，来自这些经验在状态池里的竞争。

> 原文定位：第一卷 §1 "AP 是什么"、§1.1 最短定义。

## 1. 它和 Transformer / 大模型差在哪

这是用户最关心的问题，单独在最前面说清。

| 维度 | Transformer 真/伪 | AP 真 |
|---|---|---|
| 架构 | 黑箱参数函数 | 白箱认知闭环 |
| 学习 | 海量数据一次性预训练，预训练完就**不可在线再学** | 小参数 + 持续在线学习，**每教一次就学一次** |
| 输出 | 一次性按 context 输出下一个 token | 多 tick 拟人过程：观察 → 草稿 → 回读 → 修改 → 提交 |
| 泛化 | 模式匹配（统计相关性） | 经验后验 + 范式组合（如竖式 = 事实 × 过程范式） |
| 可解释 | 注意力热图（事后解释） | 每 tick 完整审计槽可回放 |
| 情感 | 无 | 12 认知感受通道 + 情绪慢量调制表达 |
| 主观能动性 | 无（必须被问才答） | 有（未闭合张力累积到自发外显，且说一次会自己安静） |

**最关键差异**：Transformer 的"像人"是**输出像人**——因为喂了海量人话。AP 的"像人"是**过程像人**——看见不熟会先感到违和，没做完会惦记，没把握会先请教。第一 tick 就直接吐"苹果"且没有视焦点移动、采样、把握上升、草稿、回读、提交竞争，那它语气再温柔也不是 AP 拟人。

> 原文定位：§4.3 与 Transformer 的区别、§2.1 全栈拟人的定义。

## 2. AP 的完整闭环（一个 tick 干了什么）

```
外界刺激 + 内源预测
        ↓
   感受器 (receptor)         文本/视觉/听觉/奖惩/时间/内源
        ↓
   状态池 (Field)            无序认知场，实/虚能量、认知压、注意能量、疲劳
        ↓
   注意力调制                压制成熟场景，回拉违和场所
        ↓
   短期结构池 (SSP)           有序、二维、空间、时空、图结构的当前经验组织
        ↓
   召回 B                     从经验流召回到"现状认知"
        ↓
   预测 C_forward             向后："接下来会怎样"
   追溯 C_backward            向前："为什么现在这样"
        ↓
   C* 融合回灌                预测/解释包以虚能量回到状态池
        ↓
   认知感受 (12 通道)          惊、违和、合理、正确、把握、期待、压力、闭合/未闭合...
        ↓
   情绪慢量                   跨 tick 积分，调制行动阈值与探索/保守倾向
        ↓
   行动竞争                   写字/移焦点/回看/修改/提交/请求教师...按 drive 竞争
        ↓
   反馈 + 奖惩
        ↓
   写入经验流 (append-only)   ← 这是 AP 经验真相源
        ↓
   下一 tick
```

**关键设计**：v0.5 裁定——AP 的基础循环**不是**单向预测，也**不是**问题来才触发的归因检索。**每 tick 同时**运行后继预测与前因归因。B 召回提供"历史相似现状感"；C_forward 形成"接下来会怎样"的虚能量；C_backward 形成"为什么现在这样"的虚能量。当前已发生的线索中和 C_backward 产生主观原因感；后续现实中和 C_forward 产生预测闭合感。

> 原文定位：§1.3 数学形式、§200 v0.5 裁定。

## 3. 内生认知主义（AP 的哲学核心）

**内生认知主义**：认知过程本身产生的内部状态——惊、违和、合理、把握、期待、压力、未闭合、疲劳、时间感、节奏感、行动后果感——都应该作为**可被感受、可被记忆、可被召回、可被学习**的信号进入认知流程。这不是给系统加标签，而是让这些状态真正参与行动竞争与学习闭环。

传统输入-输出系统学"看到 X 输出 Y"。AP 学"看到 X 后**自己产生了什么感觉**、召回了什么、预测了什么、采取了什么行动、后果怎样、下次在类似内心状态下该怎么做"。

**关键公式**：

```
Feel_t = f_internal(Field_t, B_t, C_t, ActionTrace_t, RewardTrace_t)
Field_{t+1} += encode_as_SA(Feel_t)              # 感受回灌为下一 tick 的输入
```

这就是**内生闭环**：state → feeling → state → action → feedback → state。

**示例**：AP 写草稿"3+4=8"。回读时竖式/数量感召回产生不一致，形成**违和感**与**低正确感**。违和感提升"擦除/修改/重算"的行动驱动力，惩罚提交——直到草稿改成"3+4=7"且验证信号降低认知压。这不是"if 答案错则改"的规则，是内部认知感受驱动的修正。

> 原文定位：§3 内生认知主义、§3.1-3.5。

## 4. AP 不是符号主义，也不是规则模型

**与符号主义的区别**：符号主义预先定义符号和规则。AP 允许符号和结构从经验**涌现**。比如"你好"不是先天分词给的，而是在"你好啊/你好!/你好呀"等相似输入与预测误差中逐步成为稳定认知对象。

**与普通 Agent 的区别**：

```
普通 Agent:   LLM → tool planner → memory/RAG → action       (planner 的决策是隐藏的)
AP:           sensor → field → SSP → recall/predict → feel/emotion → action → feedback → memory
```

工具调用**不是** planner 的隐藏决策，而是行动候选；工具结果**不是**外部日志，而是行动反馈 SA。

**与专家系统的区别**：AP 可有先天规则（如防作弊护栏、奖惩源、触发接口），但规则**不是**最终能力。先天规则提供倾向；后天能力来自经验流、预测误差、共现、行动后果和调参。

**AP-Core 没有的东西**（红线）：

1. 没有"问题 → 答案"的答案表
2. 没有"图片 hash/特征 → label"的标注表
3. 没有"用户意图分类 → 模板回复"的隐藏路线
4. 没有学生侧 LLM 替 AP 生成回复
5. 没有前端伪造 tick
6. 没有 UI 直接改变 AP 认知结果

> 原文定位：§4.1-4.5、§5.1-5.3。

## 5. 视觉、听觉和内心画面（多模态如何工作）

**核心**：所有模态都是**一等公民 SA**，进入**同一套**状态池和经验流。不存在"视觉识别模块 + 文本对话模块 + 语音识别模块"三个独立流水线。

### 视觉

视觉感受器把图像切成**局部视觉 SA**（focus、周边、空间/颜色/形状/纹理），这些 SA 与文本"苹果/香蕉"在**经验共现、奖惩、预测误差**中绑定。

画面重建是**三层叠加**：
- 第一层 sensory sketch——原始采样
- 第二层 remembered / prototype overlay——同类对象记忆叠加"苹果"原型
- 第三层 prediction overlay——基于当前认知场的预测叠加

**关键**：AP 看到的不是像素，是这三层合成的"它脑中的画面"——焦点清晰，周边模糊，带注视标记。这正是 APV3 把想象画面外显的依据：**AP 呈现的不是你的原图，是它从状态池视觉 SA 重建的内心画面**。

### 听觉

听觉感受器同样把音频切成局部 SA，多次听到相似句后大脑才切出重复片段。第一次听外语只记成一团声音——这点和人很像。

### 红线 C30

只暴露 `rendered_from_state_pool_canvas=True` 且非原图的渲染。你上传的原图缩略图不许当内心画面。

> 原文定位：§16 视觉感受器与内心画面重建、§22 v0.2 细化、§17 听觉感受器。

## 6. 召回 B / 预测 C_forward / 追溯 C_backward / 融合 C*

| 信号 | 方向 | 作用 | 一句话类比 |
|---|---|---|---|
| B（现状召回） | 当前结构 → 经验流 | 找历史相似现状，给出"我之前在类似情况下见过什么" | "上次也遇到过这个吗" |
| C_forward（预测） | B 向后传播 | 形成"接下来会怎样"的虚能量 | "接下来该是什么" |
| C_backward（追溯） | B 向前传播 | 形成"为什么现在这样"的虚能量 | "是因为什么到这里的" |
| C*（融合） | B + C_forward + C_backward | 形成预测/解释包，以虚能量**回灌**状态池 | 综合出"我现在该怎么判断" |

C* 回灌是 AP 的招牌——预测与解释不是被动的，它们**主动参与**下一 tick 的状态池竞争，所以 AP 会"提前感到自己会怎么做"。

**把握感**来自 B/C 召回的支持度与退火后验（§173.5）——把握**不是**置信度阈值表，是真实退火出来的概率。

**Local Recall Upgrade (M2)**：exact 与 structural 各算 drive 竞争选源，**不**用瀑布控制流决定先用哪个召回源——这是 APV3 的关键改动。

**残差竞争召回（§56.2）**：召回的查询不是只有"你刚说的这句话"，而是**整个短期序列池**——最近几段经历按"越近权重越高"参与召回。最强的子序列组合先被解释（整体优先：如果 A+B+C 一起能唤起一段记忆，它天然压过只有 B+C 的记忆）；被解释掉的部分随即降权（像谐振吸收：被匹配的频段能量被吸走，剩下的频段就相对突出），于是剩余部分（比如 D+E）在下一轮继续唤起自己的记忆。多轮结果并列进入认知场，各带自己的支持度——后续轮次强度递减，只丰富认知不抢答。

这带来三个日常可感的效果：**上下文区分**（同一句"你好"，前文聊过下雨和没聊过，它的认知状态不一样）；**联想与灵光**（发呆时回味过 A，现在遇到 B，同时和两者有关的旧记忆会被唤起——"忽然想到"就是这么来的）；**经验越用越细**（每次解决问题的过程本身成为新记忆，下次类似情况可召回的经验就更精确）。

> 原文定位：§4 召回预测追溯与理解、§24 v0.2 数学与性能实现。

## 7. 12 认知感受通道

不是装饰，是认知闭环的**底层电流**。当前已接入工作台可见层的有：把握、期待、压力、未闭合、疲劳。其余通道按白皮书逐步接通，**不口头补齐**（白皮书明确禁止"在文档里说会了但代码里没接"）。

| 感受 | 来自 | 影响 |
|---|---|---|
| 惊 | 当前输入与预测的落差 | 把注意力拉回，促发采样 |
| 违和 | 召回不一致 / C* 解析矛盾 | 提升"擦除/修改/重算"驱动力 |
| 合理 | C_backward 与已发生线索中和 | 降低复查需求，可继续 |
| 正确 | 验证信号 | 提升提交驱动力 |
| 把握 | B/C 退火后验支持度 | 决定敢不敢写、敢不敢泛化 |
| 期待 | C_forward + 未闭合 | 决定要不要继续看 |
| 压力 | 认知压高 + 惩罚预测 | 让行动变保守、倾向回读或请教 |
| 闭合 | 任务完成信号 | 释放该未闭合项 |
| 未闭合 | 任务停在半路 | **M4 的招牌**：累积后 idle 自发外显"我还在想这个" |
| 疲劳 | 持续高压 + 长任务 | M4-2：调低 idle 节奏 |
| 节奏感 | 节奏错位 | 让 AP 暂停或主动调整 |
| 后果感 | 行动结果与预期差 | 驱动学习，写入经验流 |

**SA 回灌（M4 的另一个招牌）**：高惊感受回灌状态池为 `feeling::SA`——AP"感到自己在慌"是它能在状态池里读到自己的感受对象，这就是**元认知**的底层接口。

> 原文定位：§3 内生认知主义、§5 第五卷 闭合感、任务、情绪与行动、§187.3 idle pacing。

## 8. 情绪如何工作

情绪是**慢变量**，不是一次 tick 的标签。它会**跨 tick 积分**，决定 AP 后续更敢泛化还是更谨慎、更愿意表达还是更克制。

- **正向影响**：连续成功、被肯定、把握上升时，表达会更松，泛化会更敢一点。
- **负向影响**：连续受挫、未闭合增多、疲劳上升时，动作会变保守，更倾向请教或回读。

**招**: 没有独立"情绪模块"，情绪是认知感受和奖惩脉冲的统计慢量，调制的是行动阈值与探索/保守偏置，不是"加情感词"的设定。

> 原文定位：第五卷 §27 情绪如何工作。

## 9. 短期结构池（SSP）、统一经验流与六阶段学习协议

### 短期结构池

无序状态池里被注意力调制的对象**进入有序**短期结构池，形成**认知流**："它刚才看到的文本/视觉/行动的顺序和关系"。SSP 是 AP 的"工作记忆"。

### 统一经验流

**append-only 的只追加经验日志**是 AP 唯一真相源（LongExperienceLog）。Indexes 只是加速层，随时可重建。

- 高价值焦点片段、行动后果、奖惩、感受、视觉/听觉采样、草稿操作都写入事件流
- 记忆包是**经验证据包**，**不是**技能插件/答案包
- 卸载某记忆包 = tombstone 该批次引用的证据（软删，召回即刻失效），不动写过的 tick 历史

### 六阶段自然学习协议

AP 的学习不是"喂答案"，是从教学、复盘、自测、冷重测、退场就绪一路**慢慢长出来**。每阶段都不是 UI 标签，是经验流里的真实条件：

1. **被动观察**（passive observation）——先看
2. **回声模仿**（echo imitation）——复述 "你好啊"
3. **后继预测**（successor prediction）——"你好啊 → 你也好"
4. **多回应聚合**（multi-reply aggregation）——你好/好啊/你也是/嗯你好
5. **范式组织**（keyword organization）——你好/也好
6. **语法风格精修**（refinement）——"嗯，你好呀"

**teacher-off 是 AP 的硬指标**：不是抛弃老师，而是证明系统能在老师离开后**继续保持闭环**。教学纠正、反馈、负例抑制、自测、冷重测是这个闭环的基本组成。

> 原文定位：第六卷 §34-36、§23 v0.2 SSP + 经验流设计。

## 10. 为什么它能数学泛化（招牌时刻之一）

AP 的数学能力**不是**内置计算器（红线 §65.3 禁 eval/solver）。它的数学泛化是**"事实库 × 过程范式"**的乘法结构：

1. **事实库**（echo 阶段）：先教 AP 个位加法事实（2+3=5、4+5=9...），AP 通过正常教学写入经验流。
2. **过程范式**（范式阶段）：教师示范竖式过程——先写第一行 → 换行对齐 → 逐列加 → 进位 → 回读验算。AP 从**行动序列共现波峰**自动**发现**这套范式（不是被写在 if-else 里的规则）。
3. **组合**：未教过的组合（如 42+35）由"逐列子召回 + 进位链"组合出来——每列结果是已教事实的 exact 召回，进位是"召回结果的高位进入下一列"。**缺哪条事实就卡住，教了就通**。它会诚实说"我不会"，而不是胡乱猜个答案。

这正是儿童学习路径：先背加法表，再学竖式过程，然后能算没背过的题。

**APV3 验收标准**：教个位加法事实 + 竖式示范后，AP 能用范式组合算出未教过的题（如 42+35=77）；缺事实时**诚实**说不知道（87+96 不会装懂）；**零 Python eval**。

> 原文定位：§65 数学模型细化、第九卷、§41 学习技能。

## 11. 主观能动性：它能自己想起、自己说

AP 与一切 chatbot 的本质区别之一。它不是被问了才答，会在以下条件下**自发外显**（M4 招牌）：

- **未闭合张力**累积到阈值 → idle tick 自发说"我还在想这个。"
- 说一次后受 `repetition_fatigue` 抑制 → **自然安静**不刷屏
- 60 turn 挂机只说 1 次，不骚扰
- **无张力时保持沉默**

驱动源是**真实张力变量**（u 值累积）而非定时器；心跳节奏由 arousal/curiosity/fatigue 三者派生（M4-2）：

```
interval_seconds ∈ [2, 30]   由 arousal(↑活跃) / fatigue(↓活跃) / curiosity(↑活跃) 决定
```

**主观能动性当前评分（fable5 自评）**：7/10。方向正确、单一但真实：
- ✓ 三点涌现而非脚本：有事惦记就提、说过不烦人、心情好活跃
- ✗ 当前自发行为只有"维持未闭合"一类。人的自发还有**回味、练习、找新事**（你例 3 的完整清单）——这需要 L3 正 Q 候选生成器扩展。

**演示文案建议**：诚实说"它现在会惦记没解决的事，将来会自己找更多事做"，不许用"智能涌现"等夸大词。

> 原文定位：第五卷 §187.3、§27.1 求知欲来自认知压高+惩罚信号、M4/M4-2 协议。

## 12. 行动竞争与 DraftGrid（输出是怎么来的）

AP 的输出**不是**一次性根据 context 吐下一个 token，是**行动竞争**的结果。候选行动有：是否继续观察、是否移焦点、是否写字到 DraftGrid、是否回看草稿、是否修改、是否提交、是否主动停、是否请求教师。

**DraftGrid** 是 AP 的草稿区。AP 不是一次性写出整句，而是**逐字写入** DraftGrid → **回读** → **修改** → **提交**。数学竖式也在这里逐格写。看 tw-7_workbench → 详情页 → 草稿格可以看 AP 一格一格的逐格写入过程。

这是 AP 拟人输出的最后一步——**像在写草稿**，不是连珠炮式生成。

> 原文定位：§18 DraftGrid、§19 经验流、§5 行动如何竞争。

## 13. 范式发现与红线（防作弊）

### 范式发现

范式**不是人工写的规则**，是 AP 自动从经验流行动序列的**共现波峰**中**涌现**的：
1. 发现经验中重复出现的多动作序列
2. 提升该序列整体在行动竞争中的偏置
3. 用反例（惩罚）抑制错误范式
4. 做范式内偏执（缺事实时诚实卡住，不胡编）

### 红线（防作弊总表）

为证明 AP 的能力是**真长出来**而非作弊，APV3 设了硬红线：

- ✗ 禁止答案表 / 图片 hash→label 表 / 意图分类→模板回复
- ✗ 禁止学生侧 LLM 替 AP 生成回复（LLM 只能当教师、审计员、调参助手）
- ✗ 禁止 turingsim 伪造或前端伪造 tick
- ✗ 禁止 UI 直接改 AP 认知结果
- ✗ 禁止外部 OCR/ASR 在 teacher-off 阶段冒充 AP 感受器
- ✗ 禁止新 DB 表把"想象火躲开"和"真实火躲开"学成同一行动后果（§37 SDPL 源分化包学习）

### L3 在线嵌入

L1 现状召回 / L2 时序空间因果 / L3 行动后果——AP 自己从**预测误差、共现、结构邻近、奖惩、行动后果**中训练出的轻量向量表示。

**红线**：在线嵌入不替代白箱显式通道；不可全库暴力；必须有预算、置信、支持度、审计。

## 17. 常见质疑与回应（FAQ）

以下是 APV3 面对外部审查时最常见的质疑，以及有理有据的回应。

### Q1: "它不如 GPT 流畅，表达单调"

**回应**：AP 追求的不是统计流畅，是真实认知发展。

- GPT 的流畅来自 1750 亿参数吃了互联网全量文本——它不理解自己在说什么，只是统计上"下一个 token 概率最高"
- AP 的表达来自真实经验：教过什么就会什么，没教过就诚实说"不太会"
- 人类幼儿也是从单字→双字→整句发展，3 岁小孩不会说"综上所述"——这恰恰证明 AP 在走真实发展路径
- AP 的表达多样性随学习自然增长：范式槽 + 认知主导填充 = 组合出从未见过的句子（数学泛化已证明）

### Q2: "和 ELIZA 有什么本质区别？"

**回应**：三个可验证的根本差异：

1. **同输入异输出**：对 AP 说同一句话，第 1 次正常回复，第 4 次因疲劳习惯化选择沉默——ELIZA 永远给相同回复
2. **泛化到从未见过的输入**：教 AP "23+45=68"，它能算出从未见过的 "31+26=57"——ELIZA 做不到
3. **内部认知状态可审计**：每个 tick 的竞争、感受、情绪全部白箱记录，可追溯为什么这样决策——ELIZA 只有模式匹配规则

ELIZA 是字符串替换，AP 是认知闭环。一个是纸做的花，一个是活的种子。

### Q3: "凭什么说这是'人格'而不是状态机？"

**回应**：

- **情感真实影响决策**：疲劳→沉默、好奇→求知、犹豫→回读修改、压力→保守退缩——这些不是预设的 if-else，是 12 通道感受 + 情绪慢量在竞争中自然涌现的
- **个体差异**：两个 AP 从相同底座出发，经历不同教学后产生不同的记忆、不同的反应模式——这就是"人格"的定义
- **完型崩溃**：重复刺激 fatigue→1.0 时刺激显著性被压缩到 0，AP 对该输入完全"视而不见"——和人类看不见自己鼻子的机制相同
- **白箱可证伪**：打开 workbench 审计面板，每一个决策的来源都可追溯。状态机做不到"解释自己为什么这次选了沉默"

### Q4: "为什么不直接用 LLM？"

**回应**：因为目标不同。

| | LLM | AP |
|---|---|---|
| 定位 | 终点（生成即结束） | 起点（每次交互都是学习机会） |
| 记忆 | 无（context window 外就忘了） | 持久化经验流 + 遗忘/复习节律 |
| 学习 | 不能（推理时参数冻结） | 每教一次就学一次，实时可验 |
| 解释 | 不能解释为什么这样回答 | 每 tick 完整审计可回放 |
| 个性 | 靠 prompt 假装 | 从经验中自然生长 |
| 成本 | 每次推理消耗巨量算力 | 本地 SQLite + CPU 即可运行 |

LLM 是"看起来像人"——因为吃了人说的所有话。AP 是"过程像人"——它的每一步决策都走人类认知的路。

### Q5: "跨域泛化能力不足"

**回应**：这是发展性特征，不是静态缺陷。

- 同域精确率 10/10 证明机制正确——AP 在自己学过的领域内做到了完美泛化
- 跨域 4/6 恰恰说明它在"学习中"——人类幼儿也不是一出生就能跨域迁移
- 随着教学积累，跨域能力会自然增长——这是发展轨迹，不是能力上限
- 如果一出生就什么都会，那才是假的（那叫预训练，不叫学习）

### Q6: "系统太复杂，可维护性存疑"

**回应**：

- 认知系统本身就复杂——人脑有 860 亿神经元，AP 用 15 个模块 + 231 个测试文件实现一个可运行的认知闭环，已经是极度精简
- 991 项测试全绿，覆盖 Phase 1 到 Phase 21 全生命周期
- 每个模块有明确的白皮书章节对应（§9 状态池、§30 感受通道、§32 行动竞争…）
- 审计槽设计让任何行为都可追溯到具体公式和参数——比"一个 175B 参数黑箱"的可维护性强得多

### Q7: "有什么实际用途？"

**回应**：

- **数字伴侣/虚拟宠物**：真正认识你、记住你、随你成长的 AI 伴侣（不是每次重新读 prompt 假装认识你）
- **教育 AI 助手**：能被学生教会的 AI（学生通过"教会 AP"来巩固自己的知识——费曼学习法的 AI 实现）
- **认知科学研究平台**：白箱可观测的认知过程，用于验证认知理论假设
- **游戏 NPC**：有记忆、有情绪、会成长的非玩家角色

它是把"看见—想起—感到—决定—做—回头看—学"这套人脑闭环，做成一个真运转、可回放、可在小白双击 bat 后立刻看到 tick 里发生了什么的认知底座。它的"像人"是过程像人，不是输出像人。

你觉得它"不够聪明"？那就教它。你教的每一句话都会变成它的经验——这不是 prompt，这是记忆。
