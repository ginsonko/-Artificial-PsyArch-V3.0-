# APV3 Phase 20.5a3 Workbench Memory + State-Pool Replay Repair Final Report

## Scope

Phase 20.5a3 fixes the workbench issues raised during browser testing:

- Current-session user text is visible in chat bubbles.
- User text is observed into AP-native local memory as `user_observed_utterance`.
- Ordinary user utterances are not allowed to become reply templates.
- Teacher corrections remain the only phrase kind eligible for taught-response recall.
- Runtime replay remains per-tick AP loop snapshots, not a staged fake pipeline.
- Inner picture, thought cloud, and audit panels render from `RuntimeTickEvent` / `state_pool_top12`.
- The inner picture explicitly marks state-pool reconstruction and visual focus, not original-asset replay.
- Audit panel exposes 17 per-turn/per-tick charts, including backend process timings.

## Design Review Decisions

1. User input must enter AP memory in the isolated Phase20 workbench DB. This replaces the older test assumption that unknown user text must never appear anywhere in state.
2. Reply learning still cannot be a direct mapping table. `user_observed_utterance` is visible/exportable memory, but `_select_taught_response()` filters it out. Only `teacher_event_cooccurrence` / `teacher_phrase::...` can become a taught reply.
3. UI panels are views of runtime data. `inner_picture_state.layers`, focus, state-pool energy, and `audit_metrics` are generated by the turn runtime and rendered by the browser.
4. Hearing remains bounded: the UI says the auditory receptor is not enabled when no audio SA exists; it does not pretend TTS is inner voice.
