最小 CLI 体验入口

现在可以直接在本地运行：

cd APV3.0test
python -m apv3test.chat

可用命令：

输入中文：运行一轮 AP 对话 :top：查看当前高支持短表达 :mode：切换结构态 :+：轻微奖励上一轮输出 :-：轻微惩罚上一轮输出 :quit：退出

CLI 命令只是外部体验壳。学生侧普通输入仍然走 AP-native 统计、共现、内省 feeling 和 style gate，不按用户文本写答案分支。

一轮输入如何变成输出

1. 用户输入作为外部 query 进入当前结构态，不做关键词回答。
2. 词库匹配只有完全匹配 seed phrase 的短表达才成为共现证据。
3. 内省 feeling由结构态 view 触发当前认知感受标签。
4. 共现召回从 feeling 到 phrase 的后天关联中选候选表达。
5. 风格门输出经过 style gate，保存到 SQLite。

结论

Phase8.0 完成了从“机制已经在测试中成立”到“可以干净启动、连续输入、保存重启”的工程化第一步。它保留 APV3 的核心边界：后天共现学习、固定安全词库、无学生侧 LLM、无答案表、无测试数据库污染。

这意味着项目已经具备最小真实体验条件。下一步可以在这个入口上进行真实用户试玩观察，再决定是否优先扩词库、做 Web UI，或加入更细的 context 条件化表达。
