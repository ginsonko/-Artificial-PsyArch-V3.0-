## 〇、终极目标与四阶段路线图

### 终极愿景（逻辑根基）
用**内生认知主义**路线造出一个可解释（白箱）、可在环境中持续在线学习、拟人、不依赖大参数预训练的**通用心智**。AP 路线主张：**能力从内部信息流闭环中涌现**，而非靠数学拟合海量参数。

> 一个能像人一样：感知→维持认知场（状态池能量）→双系统召回预测→产生认知感受与情绪→行动竞争→从预测误差/共现/奖惩中持续学习→自适应环境稳态，且**全程白箱可审计、可回放**的人工心智。它不是更聪明的 chatbot，而是一条通往 AGI 的**结构性替代路线**——"通过这套系统在真实环境中记录和统计信息，达成比数学拟合更灵活、更符合实际的效果"。

### 四阶段路线图

| 阶段 | 目标 | 与理论模型的依赖 | 状态 |
|---|---|---|---|
| **① 开放对话底座** | 持久中文自由对话：真 runtime tick、状态池能量循环、双系统召回预测、DraftGrid、行动竞争、多 tick 观察。**文本模态认知闭环端到端跑通**。 | 九大模块的文本子集 + 能量循环 + 双系统召回 + 输出管线 | **当前进行中（最近）** |
| **② Agent / 通用智能体** | 在对话底座 + 行动/驱力系统上，AP 能接任务、用工具（注册行动器）、经行动评估+奖惩（海豚训练式）学习"什么场景该做什么"，在应用中变得有用。 | 行动评估（召回相似状态+行动成功经验→预测 C*→评估奖惩虚能量→影响因子）+ 行动后果在线学习嵌入 L3 | 未开始 |
| **③ 桌宠（StrongestNurturingSystem）** | 产品壳：活在用户桌面、感知桌面、有性格、有可视化（状态池气泡/能量曲线/审计折线/想法云/内心画面）。**AP 面向人的具身载体**。产品逻辑 ≠ 认知逻辑。 | ①②认知核心 + 桌面感受器/执行器 + 视图层 | 部分（壳已有，认知未接通） |
| **④ 具身智能** | 注册物理执行器（电机）、感知物理世界、具身操作。 | ①②③认知核心稳固后 | **暂不做** |

### 各阶段"要的效果"
- **①**：AP 不是一问一答分类器。输入一句话/一张图，它要像逐步看的人——会观察、犹豫、召回、在 DraftGrid 里一个词一个词写草稿、停顿（标点处后继偏置分布平均）、回读修改、然后提交。教学纠正后**能学**（共现+信用分配，非写答案表）。苹果教成苹果、香蕉教成香蕉后，再看苹果**不会变香蕉**（视觉 SA 差异+共现，非最近答案）。
- **②**：AP 像海豚——你（环境/人工/LLM）通过奖惩教育它"这个场景这样做对、那样错"，它经**行动评估**自己学到"在什么现状下做什么行动最可能得奖"，从而在应用中**自主表现出有用行为**，这是统计学习而非硬编码。
- **③**：AP 以桌宠形态长期陪你，性格/情绪/注意力随交互演化，有可读的内心可视化。它是 AP 的**体验载体**和**长期养成对象**。
- **④**（暂缓）：AP 能驱动物理身体在真实世界行动并感知后果。

### 与 transformer 路线的本质区别（哲学根不能让步）
transformer 靠"海量数据拟合一个参数函数"；AP 靠"小参数 + 在真实环境里持续记录统计 + 预测误差自学习"。前者黑箱、不能在线学、一次出答案；后者白箱、能在线学、多 tick 拟人过程。**任何引入"学生侧 LLM/隐藏求解器/答案表/标签表伪装成 AP"的做法都破坏根基。**

### 对最终目标的逻辑可推导性
- **→①对话底座**：能量循环 + 双系统召回预测 + DraftGrid + 行动竞争跑通 → 多 tick 自由对话涌现（非一次分类）。✅ 数学闭环自洽。
- **→②Agent**：行动评估 + 行动后果在线学习嵌入 L3 → AP 统计学到"什么现状做什么行动最可能得奖" → 应用中自主表现有用行为。✅ L3 嵌入是关键。
- **→③桌宠**：①②认知核心 + 桌面感受器/执行器 + 视图层 → 桌宠形态。✅ 视图只显示 RuntimeTickEvent。
- **→④具身**：注册物理执行器。✅ 行动器抽象已支持（神经网络/插件/物理结构都可注册）。
- **→AGI/通用性**：自适应调参器（任何环境稳态）+ 在线学习嵌入（任何模态/任务自学习）+ 最小预测误差自学习 → **结构性通用**，不依赖预训练。✅ 这是路线成立的理论根基。

## 一、九大模块（修正版）

```
感受器 → 状态池(认知场) → 数据库(记忆) → 注意力(滤波+能量增益)
   → 先天编码 → 认知感受通道 → 情绪通道 → 行动器与驱动力管理 → 自适应调参器
```

1. **感受器**：外界刺激→最小可分辨单位 **SA 列表**→外源性刺激。视觉（轮廓/色彩/亮度/采样/空间位置，**采样按视焦点与边界偏置**）、听觉（频率/响度/波形）、文本（字符/短期排序）。**每种都应可重新还原/组合成想象画面**（内心画面/内心声音的来源）。⚠️ 感受器只产 SA，不做语义判断。
2. **状态池（认知场）**：不是容器，是**能量动态平衡系统**（详见第二节）。背景认知 + 能量分布 + 认知压。
3. **数据库（记忆）**：两类本体——**状态池快照**（低可读）+ **注意焦点记忆**（高可读）。支持内容相似度通道、时空邻近通道、能量/认知感受/情绪通道加权召回。⚠️ 召回是**生成虚能量预测场**，不是"取回复候选"。
4. **注意力**：滤波器——增益波峰、抑制杂音。⚠️ **注意力消耗资源，对状态池加能量增益**——这是内源性持续演化的关键（没它，无外源时能量只减不增）。注意焦点→叙事化想法（生活化短句、碎片词汇、跳跃思维，**非**严谨长文；要严谨长文需注册"文本编辑器"行动器，AP 像人逐词写、回读、增删改、满意再发）。
5. **先天编码**：**最关键模块**，决定 AP 像猫/虫/人。触发认知感受/情绪/行动，塑形发展方向。先天+后天赋能双触发：感受/情绪/行动节点是**一等公民 SA**，被召回赋能时重建通道变化。
6. **认知感受通道**：**通用法则**——一切可量化向量通道（时间/空间方位/听觉波形/响度/能量值/变化量/变化率/全局统计/疲劳…）超阈值→赋能→记锚点。违和/正确/惊/合理/把握/期待/压力都是认知感受。是"自我认知"的底层。
7. **情绪通道**：慢量调制——注意力集中度/注意力资源/全局行动阈值/奖惩最终赋能系数。6~8 调制因子，可含疲劳代偿。状态量本身可解释、可白箱。
8. **行动器与驱动力管理**：驱动力管理器（叠加/衰减/超阈值触发/冲突竞争）+ 复数注册行动器。**行动评估**是学习核心。⚠️ **坚决程度=驱动力−阈值**，作行动效果参数（如注意力聚焦坚决度）。
9. **自适应调参器**：最大正常范围/理想运行范围，超范围急调，区间内低优调，调坏回滚。实现"通用性"——任何环境都能逐步习惯适应到稳态。**阶段②③的通用性根基。**

## 二、单 tick 能量循环（AP 的"心脏"，数学主干）

### 每 tick 能量流动（后续所有实现都得对得上）
```
[1] tick 开始：所有 SA 能量 ×衰减系数（乘算）
[2] 外源性刺激 SA 注入实能量（加算）
[3] 注意力消耗资源，对波峰 SA 增益（加算）
[4] 注意力调制后的状态池 → 分两路：
      路A【状态池采样信息流】→ 快系统召回
      路B【注意焦点信息】    → 慢系统召回
[5] 各路召回 B/B'（继承状态池能量×效率因子×相似度权重）
[6] B/B' 再召回 C/C'（C 获虚能量赋能）
[7] 所有 C 叠加成唯一 C*（预测包）
[8] C* 虚能量按 SA 粒度回灌状态池（加算，同 SA 合并）
[→] 稳定外源输入下，趋于动态平衡（最小预测误差）
```

### 核心量与符号（必须严格区分）
- **R 实能量**：外源刺激 + 先天规则/行动产生的"确实发生"
- **V 虚能量**：预测与感应赋能的"预测应当发生"
- **认知压 = R − V**：大小=实虚错配程度；**R>V（正）=预测不足/惊**；**R<V（负）=过度预测/违和**。⚠️ 符号有意义。
- **匹配效率因子（把握感）**：与最高相似度正相关。内容+能量分布完全一致=1.0，重合越少越低。**避免低质量记忆被认成现状**。
- **B/B' 能量分配**：B 总能量 = 状态池采样/焦点总能量 × 效率因子，按各 B 归一化相似度权重分配。**B 不回灌状态池**，只用于召回 C 并赋能。
- **C/C' 虚能量**：总量 = B 能量 × 效率因子，叠加成 C*。**只有 C* 回灌状态池**。

### 能量图景（环境差异化，拟人"情绪状态"的能量来源）
- **熟悉稳定**：R≈V，快系统自然行动，注意力放松
- **不熟悉**：把握低→V 低→R 主导→"惊"大量→注意力聚焦→慢系统代偿→紧张集中
- **熟悉遇意外**：部分过拟合（V>R→违和）+ 部分欠拟合（R>V→惊）→预测与错配对象同入注意力→建立同义/跨上下文/对立/多可能联系

## 三、双系统召回 + 记忆本体 + Tick 记忆落地设计

### 3.1 双系统（"快/慢"是召回系统，不是记忆层级！）

| | 快系统（认知场召回） | 慢系统（焦点召回） |
|---|---|---|
| 匹配对象 | 整个状态池采样（数百~数万 SA） | 注意焦点（数个~数十 SA） |
| 能量分布 | 均匀，多数 SA 能量少 | 高能，纯度高，受杂音少 |
| 产物 B | Bn（现状认知） | Bn'（注意认知） |
| 产物 C | Cn，主为状态池快照，**支持并行多行动/情绪/认知节点** | Cn'，主为注意焦点记忆，**易成下 tick 注意对象→连续后继偏置** |
| 人类类比 | 熟能生巧的快速反应（不需注意力决策） | 连续清晰的内心推理/续写 |

**关键**：慢系统发展到结论，Cn' 聚合体在状态池占主导，下 tick 快系统可直接以这些 Cn' 为现状认知继续，不必每步走慢系统——**这就是推理"自动性"的来源**。

**后继偏置**：N-gram 式但被预测场调制。后继偏置有唯一/明显优势解时主导；分布平均时每个可选项能量低，状态池预测场主导。输出像人——深思、逐词、可回读修改、标点处会"停顿"。

### 3.2 记忆本体论（两类，不是两层级）
1. **状态池快照记忆**：特定 SA 分辨率下的能量分布 + SA 内容。低可读性。快系统召回对象。可还原叠加为 C*。
2. **注意焦点记忆**：每 tick 注意焦点信息（最热少量对象）。文本 SA→"词汇/短句"；图像 SA→"画面对象"（苹果轮廓+颜色+采样→苹果画面记忆）。高可读性。慢系统召回对象 + 后继偏置链。**前提：曾同时进入注意焦点才视为整体**；内部顺序不敏感，但时间顺序通道决定先后（实际顺序/平均赋能时间加权）。

> 🔑 **关键对应**（解决"12 快 12 慢"映射）：
> - **"快记忆" = 状态池快照**（快系统召回对象）
> - **"慢记忆" = 注意焦点记忆**（慢系统召回对象 + 后继偏置链）
> - 每 tick：**1 条快照 + 1 条焦点记忆**。12 tick = 12 快照 + 12 焦点记忆。

### 3.3 ⭐ Tick 记忆落地设计

**结论：原始记忆原子 = 每 tick 1 快照 + 1 焦点记忆；跨 tick 的"概念"（苹果）由在线学习嵌入层从统计中涌现，绝不预物化成概念表。**

**为什么不能预物化概念表：**
- 概念表（图片特征→label、answer 表）是红线，破坏 AP 哲学——概念应**涌现**而非**手建**。
- 人类也不存"苹果"原子；人存大量"苹果 episode"，"苹果"是 episode 簇的统计吸引子。

**为什么必须每 tick 存（不能只存聚合）：**
- **后继偏置依赖焦点记忆链**：后继偏置是 N-gram 式，作用在"上一 tick 焦点→下一 tick 续写"的链上。没有每 tick 焦点记忆，后继偏置就断了。
- **跨 tick 学习需要原始 episode**：共现/语序/因果都是统计原始 episode 得来的，丢了 episode 就丢了学习原料。
- **可回放/可审计**：每 tick 记忆是白箱的基础。

**跨 tick 概念如何涌现（三层）：**
1. **共现统计层**：相似焦点记忆反复共现，`CooccurrenceAssociationStore` 自然聚出关联。
2. **在线学习嵌入层（点5，必须落地）**：把反复共现的焦点记忆在嵌入空间拉近→形成概念吸引子。**这才是"苹果概念"的真正归宿**——它不是一条记录，是嵌入空间的一个簇，被召回时通过相似度命中。
3. **时空邻近层**：状态池快照按时空邻近召回 C*，自然带出"上次这种状态之后发生了什么"。

**落地为两张原表 + 三层涌现（替代当前三张错表）：**
```
原表（每 tick 写入，不可变，白箱基础）:
  state_pool_snapshots   ← 快系统召回对象 = "快记忆"
  focus_memories         ← 慢系统召回对象+后继偏置链 = "慢记忆"
涌现层（统计/学习产生，非物化）:
  共现边 / 在线学习嵌入（概念簇）/ 时空邻近 C*
```

**UI/记忆包呈现（bug 6 的正确修法）：**
- 统一"本地记忆/记忆包"一处展示，分两栏：
  - **注意焦点记忆**（主，高可读）：显示文本/画面含义、支持选择性删除。
  - **状态池快照**（辅，可折叠）：显示能量统计摘要，不逐 SA。
- 删除一条焦点记忆 = 删一 tick 的焦点组合；删除一条快照 = 删一 tick 能量分布；删除后 B/C 召回不再命中——AP-native 删除。

## 四、学习机制（含在线学习嵌入 L1-L3 必须落地）

### 4.1 三种学习原料（都喂进统计/在线学习，非写答案）
- **共现/相似度**：source_sa ↔ phrase/对象 加权、tick 衰减
- **奖惩 + 信用分配**：eligibility trace；纠正 = 带来源标签 teacher event
- **预测误差（认知压）**：R−V 驱动召回调参

### 4.2 ⭐ 本地在线学习嵌入（最优嵌入，必须落地，非可选）

> 哲学：纯哈希嵌入丢失语义（同义改写、长距离依赖难）；纯学习嵌入黑箱。**最优 = 白箱统计记忆 + 本地多层在线学习嵌入**。训练数据来自 AP 自身的最小预测误差/共现/错配/奖惩。

**至少三层（可按任务扩展）：**

| 层 | 训练数据 | 学习目标 | 召回用途 |
|---|---|---|---|
| **L1 Bn 召回准确性层** | 认知压（R−V） | 正压→拉近当前 SA 与实能量对象；负压→拉远 | 下次 B 召回按嵌入距离分配权重→**最小预测误差自学习** |
| **L2 时序/因果层** | 文本语序/音频先后/视觉运动趋势 | 非对称关系（顺序/因果/空间趋势） | 区分"狗咬我"vs"我咬狗"；正确顺序召回 |
| **L3 行动后果层** | 奖惩反馈 | 行动场景→后果（奖/惩）预测 | 阶段②行动评估的核心——AP "自觉性"预测行动后果 |

**关键性质**：即使纯哈希嵌入，靠**认知闭环**也能在理论上达成学习嵌入的效果（预测与现实共现→隐含语义；持续专注演化→长距离依赖；教学指认→概念）。但加在线学习嵌入是**冗余最高、效果最好**的实现。这是阶段①就要埋、阶段②③成熟的能力。

## 五、六阶段学习协议 + SDPL（源分化包学习）

### 5.1 ⚠️ 两套"阶段"必须分清
- **六阶段学习协议**（Language Learning Ladder）= 语言/技能**发展阶梯**，权威源 `EDUCATION_PROTOCOL.md:858-868`
- **APV3 Phase 编号**（Phase 20/20.6…）= **工程里程碑**，不是学习阶段
- **教师退场曲线**（Scaffold Fade Schedule）= 教师干预强度的 6 级衰减，与六阶段正交

**Phase 20 开放对话底座是承载六阶段协议在线跑通的 runtime 容器，不是六阶段里的某一阶段。**

### 5.2 七级发展阶梯（含 Stage 0）
```
0. passive observation / social sensing   被动观察/社交感知
1. echo imitation                          回声模仿
2. successor prediction                    后继预测
3. multi-reply aggregation                 多回应聚合
4. process-paradigm binding                过程范式绑定
5. keyword organization / focus_slot_filling  关键词/焦点槽位组织
6. grammar and style refinement            语法风格精修
```
（APV3.0test `curriculum.py:13-20` 把 Stage 5 特化为 `focus_slot_filling`，Stage 6 改为 `recall_only_validation`）

### 5.3 每阶段目标/触发/数学机制
| 阶段 | 目标 | 触发 | 数学机制 |
|---|---|---|---|
| 0 | 观察他人行为语言，积累材料 | 启动/无召回场景 | 状态池积累（无输出压力） |
| 1 | 把感知语言绑定到低粒度输出动作 | 教师给短句；无清晰后继峰 | type_char 动作 + echo_success + 共现 |
| 2 | 学"一句常跟另一句" | 短语 A→B 反复共现 | 后继偏置 + lag-PMI |
| 3 | 同输入教多种合法续句，重叠片段能量聚集 | 同输入多次不同合法续句 | 多候选能量叠加 |
| 4 | 语言片段绑定 AP-native 过程锚（非外部关键词） | 认知感受+情绪就位 | RPE 校正 + process paradigm |
| 5 | 高能片段组织成短草稿 | Stage 4 通过 | focus_slot 填充 |
| 6 | 精修语法/语气词/礼貌/连贯 | Stage 5 teacher-off 通过 | grammar/style refinement |

### 5.4 递进与验收
- **Stage 5 必须在 teacher-off（教师完全静默）通过才算学成**（`EDUCATION_PROTOCOL.md:593-595`）。
- 各 domain 异步：一域可在 Stage 4 时另一域还在 Stage 2。
- 允许跳级路径：未知词 1→2→4→5；重复疲劳 2→4→5；平静陪伴 3→4→6。
- 每课程最低要求：每个常见输入 ≥3 回应变体、同过程 ≥3 不同输入、≥1 延迟召回、≥1 重复/疲劳测试、≥1 无证据诚实测试。

### 5.5 教师退场曲线（6 级，与阶段正交）
```
demonstrate → strong_scaffold → weak_scaffold → feedback_only → teacher_off → cold_retest
```
- demonstrate：state_items + action_biases + feedback 全开
- feedback_only：不给 state/action hint，**只奖惩结果**
- teacher_off：教师**完全静默**（state_items=[], action_biases=[], feedback={}），AP 只靠自己的状态池/短记忆/学到的反馈/行动后果迹
- cold_retest：teacher 仍 off，用全新/部分重置上下文，测保持

### 5.6 海豚训练式 vs 自主统计学习
- **海豚训练式（有教师示范/奖励）**：demonstrate/strong_scaffold/weak_scaffold/feedback_only → 对应 Stage 1-4
- **自主统计学习（teacher-off）**：Stage 5 必须在 teacher-off 通过；Stage 0/6 后期接近自主

### 5.7 ⭐ SDPL = Source-Differentiated Packet Learning（AP 第一原则级机制）
**所有学习按 packet_key 累积，不按 content key。** 解决"同内容异态学习"——想象火躲开→违和，看到火躲开→奖励，内容相同但来源不同，必须分别学习。

- **EpistemicSource 5 族**：PERCEIVED（外部感受器）/ IMAGINED（内源链）/ HEARSAY（听用户/他人陈述）/ REMEMBERED（从 long_term 召回）/ INFERRED（内部推理，Phase 11 启用）
- **packet_key 量化分桶**（v13.1 修了丢能量漏洞）：`packet_key = (content_with_bucket, source_with_bucket, dominant_source, feeling_with_bucket)`，与交接文档 `packet_key = sensory_feature_signature + epistemic_source + substrate + receptor_version` 一致
- **SDPL 学习规则**：共现学习/lag-PMI/行动后果**全部按 packet_key 范围**累积
- **拟人涌现**：真实看火+躲→奖励→Q↑；想象火+躲→违和→Q↓；想象火+检查→中性→Q↑。不加新公式自然涌现
- **红线**：禁止 vocab SA 加 is_real/is_imagined 字段；禁止学习规则按 if MarkerKind.X== 分支；禁止预装"现实感"为固定权重
- **教学 = HEARSAY proposition + trust_promoted gate + RPE 校正**，全程走 SDPL，绝不绕过

**SDPL 与六阶段关系**：SDPL 是六阶段全程的底层引擎，不是阶段本身。六阶段是"教什么/怎么递进"，SDPL 是"不管哪个阶段，所有学习都按 packet_key 累积"。

### 5.8 与三大机制的关系
- **预测误差/认知压驱动调参**：Stage 1-4 的 AP-native 驱动力（非阶段本身）。feeling::surprise/low_grasp/evidence_gap/dissonance 驱动 Stage 1 open_learning_focus 和 Stage 2 successor 绑定。
- **在线学习嵌入 L1-L3**：承载六阶段全程的物理层。L1/L2/L3 是 SDPL packet 写入的向量基底。
- **状态池快照 vs 注意焦点记忆**：直接对应 Stage 5 focus_slot_filling——验收 = focus_tokens（注意焦点）vs emitted_tokens 的对齐。

## 六、三种"压"的澄清

| 压 | 定义 | 锚定 | 处置 |
|---|---|---|---|
| **认知压** | R − V | 状态池内 SA | **保留**。驱动召回调参与情绪/认知感受 |
| **期待/压力** | C 中奖惩虚能量产生 | **B 对象**（现状认知，不在状态池） | **保留**。每 tick 验证（B 实能量升降），期待产奖励/压力产惩罚，与衰减动态平衡→趋利避害、主观能动性核心 |
| **runtime_pressure（当前代码）** | 草稿长度驱动 | 草稿/前缀 | **废除为独立本体，吸收为认知压** |

**理由**：`runtime_pressure` 是工程臆造（草稿到一定长度就想提交），本质用长度信号冒充认知闭合，**违反 AP 哲学**。提交时机应由"认知压趋于低 + 把握感高 + 期待验证"决定，而非草稿长度。

## 七、先天编码 / 认知感受 / 情绪 / 期待压力 / 时间感受 / 行动评估 / 自适应调参器

- **先天编码**：最关键模块。先天触发 + 后天赋能双通道；感受/情绪/行动节点是一等公民 SA，被召回赋能时重建通道。短期后天可压制先天（行动竞争/拮抗/主动回忆开心事），长期先天塑形（惊/违和/合理→求知欲；期待/压力→趋利避害）。
- **认知感受**：通用法则——超阈值→赋能→锚点。违和/正确/惊/合理/把握/期待/压力 + 能量值/变化量/变化率/认知压/全局统计/疲劳……都是。自我认知的底层。
- **情绪通道**：慢量调制注意力集中度/资源/全局行动阈值/奖惩赋能系数。6~8 因子，可含疲劳代偿。可解释白箱量。
- **期待/压力**（见第六节）：锚 B，每 tick 验证，期待→奖励、压力→惩罚，与衰减动态平衡。期待/压力 SA 可召回（认知到"我在期待什么"）、可偏置注意力。趋利避害与主观能动性核心。
- **行动评估**（阶段②核心）：召回相似状态+该行动成功经验（复用快系统库，仅筛含行动成功+反馈快照）→预测 C*→评估其中奖惩虚能量→影响因子（正增益/负减益/默认中性习惯触发）。**海豚训练式教育接口**。
- **时间感受/节奏感**：召回 B 时间差集中→间隔波峰→时间感受 SA 赋能→间隔通道加权（间隔大范围宽平、间隔小范围窄陡）。节奏感来自后继偏置+顺序通道+疲劳恢复平衡，**注意防无限循环**。
- **自适应调参器**：最大正常范围/理想运行范围，超范围急调，区间内低优调，调坏回滚。实现"通用性"。

## 八、输出管线（绝不绕路）

```
RecallCandidate(来源标签/support/priority) → ActionCandidate(drive/evidence_tags)
  → 驱动力管理器: 叠加/衰减/超阈值触发/冲突竞争(次高降到阈值下,全部降该差值)
  → 坚决程度 = 驱动力 − 阈值  → 行动效果参数
  → DraftGrid(逐 cell 写入) → commit_reply(读 grid.visible_text())
  → TTS(执行器,只读已提交文本,非内心音频)
```
- `commit_reply` 读 DraftGrid，**从不** `reply_text = taught.response_text`
- `stop_generating` 是独立 action 候选（主动停，由认知压/把握感驱动，**非草稿长度**）；系统超时/crash = `system_boundary`（非 AP 主动停）
- TTS = 本地执行器，**非 inner_voice**；inner_voice 是听觉想象/草稿感受

## 九、拟人优先原则 + 来源纪律 + 红线

### 拟人优先
- 视觉 unresolved 时先观察、移动焦点、多 tick 累积，再据召回+行动竞争写草稿——**不第一 tick 就写答案**
- Phase 21：全图显著性 → NMS → 候选注入状态池 → 扫视调度 → IOR（30-tick 半衰）→ 每 focus 局部 patch 识别 → 数感自动数
- 内心画面从状态池视觉感受器采样重建：焦点附近清晰（高斯 `exp(-d²/2σ²)`），远处模糊/稀疏，体现形状/纹理/颜色/亮度/边缘通道，**且应随焦点移动累积新清晰采样**
- 标点处后继偏置分布平均→每个可选项能量低→"停顿"；输出像人逐词、可回读修改

### 来源纪律
- 一切 SA 带 `epistemic_source / substrate / receptor_version`
- `R_sketch` 是 perceived；remembered/prediction 必须分层
- 心智理论来源边界：`OTHER_OBSERVED ≠ SELF_EPISODIC ≠ IMAGINED_PERSPECTIVE`（混淆时 self_other_boundary→0）
- 视觉教学不读文件名/路径/标签，只走视觉 SA + 教师文本共现
- **允许的外部角色**：感受器、执行器、缓存、视图、教师来源、安全边界——**不能成为语义权威**

### 红线（破坏哲学，禁）
关键词硬门/正则路由；文件名/路径/测试标签进语义；图片特征→label 表；整句回复模板直接执行；学生侧 LLM/隐藏求解器/OCR 伪装 AP；UI 自己生成决策/假 tick/流水线伪装逐 tick；TTS/浏览器/向量库决定语义答案；把 APV3.0test 当 APV2.1 补丁；**草稿长度驱动停/提交**；**预物化概念表**。

## 十一、六个 bug 的根因与理论修法

### 核心结论
**6 个 bug 不是 6 个独立前端问题，而是同一组底层数学模型缺失（状态池能量循环 + 双系统召回 + 在线学习嵌入未实现）在前端的不同投影。修前端是治标，必须先重做底层。**

### bug 1：永远"嗯,听着。"，教学不生效
**根因链：**
1. `_state_items_for_tick` 每 tick 构造**静态写死能量** SA 快照（`phase20_6_runtime.py:637-642`：user_text 固定 0.42/0.36/0.2、context 固定 0.55/0.5/0.25、source_candidate 固定 source.support…）。状态池没有能量循环，是死的。
2. 教学记录走共现 + penalty 表（`phase20_open_dialogue.py:1202-1257`），**与状态池能量完全脱节**。
3. token 候选里 `taught` 的 support 默认很低，`_weighted_recall_labels` 在纯文本时只给 context_signature 权重（`phase20_open_dialogue.py:1389`），而 styled/runtime 候选总是存在。
4. `_build_token_candidates` 里 styled 候选 priority 0.55、support 0.5（`phase20_6_runtime.py:374-376`），几乎总是赢过 support 低的 taught。
5. 结果：**styled "嗯,听着。" 几乎总是赢**，教学纠正写了共现但召回门槛 + styled 永在，压不住。

**理论修法（治本）：**
- 做状态池能量循环（第二节），让认知压真正驱动召回——视觉/教学内容进入状态池后，会通过能量和共现影响后续 tick 的召回偏置。
- 做真双系统召回（第三节），让 taught 作为 B 候选进入能量竞争，而不是和 styled 在同一张文本候选表里比 support。
- 让 Stage 1-2（echo imitation + successor prediction）协议真正在开放对话 runtime 里跑——教学纠正的本质是 echo→successor 学习，当前 runtime 没有阶段感知。
- 短期止血（治标）：在底层未重做前，可调 styled 候选 priority/support 让其低于 taught，但这只是补丁。

### bug 2：内心画面焦点不累积
**根因：** `_sketch_samples_for_tick`（`phase20_6_runtime.py:935-1002`）每 tick 从 object_view 的**同一预计算 sketch** 读 samples，clarity 按当前 focus_xy 算高斯衰减——但**没有对源图在新 focus 处重新采样**。focus_xy 每 tick 在变（`_focus_for_tick:782-823` 循环 6 位置），但采集的清晰区不跟着移动累积。

**理论修法：** 每 tick 按**当前 focus_xy 对源图重采样**（视觉感受器 foveated sketch），把新清晰区作为新的注意焦点记忆条目累积进内心画面；远处区域保留之前 tick 的低清晰度采样。这正是"焦点附近清晰、随焦点移动累积丰富细节"的拟人重建。

### bug 3：音色（已解决）
当前 `speakIfRuntimeRequested`（`phase20_6_workbench.js:841`）严格由 runtime 的 `reply_tts_request.selected_this_tick` 触发，xiaoyi 只作偏好音色提示串，不是假云端音色。✅ 这波修改符合 AP 边界。无需理论修法。

### bug 4：状态池 top12 框未占满卡片
**根因：** 纯 CSS（`phase20_6_workbench.css`）。状态池列表容器未撑满父卡片高度。
**修法：** CSS 修 `#statePoolList` 容器 `height/min-height/flex` 撑满父卡片。

### bug 5：想法云静止散点不聚集排斥
**根因：** `_thought_cloud_items`（`phase20_6_runtime.py:1140-1141`）的 `x_hint/y_hint` 用 `(index*37/53)%100` 静态散布，幅度 ±0.42——**确定性的固定散点**，没有中心吸引、没有相互排斥、没有物理。
**理论修法：** 做力导向布局——中心吸引力（让想法汇聚中间）+ 相互排斥力（避免重叠）+ 能量驱动（attention_energy 决定大小/亮度、real_virtual_balance 决定色相）。每 tick 重算位置（或前端 requestAnimationFrame 物理模拟）。这才是"汇聚中间、相互排斥开"的拟人想法云。

### bug 6：快/慢记忆未与本地记忆统一 + 本体错
**根因：**
- 当前 `phase20_6_fast_action_chains`（存动作链）、`phase20_6_slow_memory`（存来源证据）、`phase20_6_tick_memories`（tick 级）三套分离，与"本地记忆/记忆包"是第四套 UI。
- 本体映射错：fast=动作链、slow=来源证据，都不是理论上的"状态池快照/注意焦点记忆"。
- 每 tick 一快一慢的数量对了（`_tick_memory_records_for_event` 每 event 产 2 条），但存的内容是 action_kind+draft_hash / source_id，不是快照/焦点对象。

**理论修法（按第三节）：**
- 重构为两张原表：`state_pool_snapshots`（快照，每 tick 一条）+ `focus_memories`（焦点记忆，每 tick 一条）。
- 概念（苹果）由在线学习嵌入 L1 涌现，不预物化。
- UI 统一"本地记忆/记忆包"一处，分两栏：注意焦点记忆（主，可读，可删）+ 状态池快照（辅，可折叠，能量统计）。
- 删除一条焦点记忆 = 删一 tick 焦点组合，删除后 B/C 召回不再命中——AP-native 删除。

## 十二、数学模型重设计路线图（高层，不写代码）

三块一体打包重做：**能量循环 → 双系统召回 → 在线学习嵌入**。

### 为什么必须打包
能量循环产生认知压 → 认知压驱动召回调参（L1）→ 调参喂在线学习嵌入 → 嵌入反过来影响下次召回相似度。三者是闭环，分开做必然逻辑断裂。当前实现的最大教训就是：能量循环缺失导致双系统召回只能压扁、Q表只能被旁路、在线嵌入无从落地。

### 重做顺序与依赖

**第 1 块：状态池能量循环（基座，必须最先）**
- 交付：SA 带 R/V 能量；每 tick 衰减(乘算)+外源注入(加算)+注意力增益(加算) 的动态循环；认知压=R−V 作为一等公民 SA；为后续 C* 回灌预留接口（C* 在第 2 块实现后回灌）。
- 验收：稳定外源输入下能量趋于动态平衡；认知压符号正确（R>V 正、R<V 负）；白箱可审计每 tick 能量演化。
- 替换：`_state_items_for_tick` 的静态写死能量。

**第 2 块：双系统召回 B/C（依赖第 1 块）**
- 交付：快系统（召回状态池快照）+ 慢系统（召回焦点记忆+后继偏置）两条召回路径；B/B' 继承状态池能量（×效率因子×相似度权重）；C/C' 获虚能量赋能；所有 C 叠加 C*；C* 虚能量回灌状态池。
- 验收：召回是生成虚能量预测场，不是取文本候选；虚能量回灌让状态池在无外源时也能持续演化；后继偏置链在焦点记忆上连续。
- 替换：`_select_taught_response` 的"选 taught 文本候选"压扁线。

**第 3 块：在线学习嵌入 L1-L3（依赖第 1、2 块）**
- 交付：L1（认知压驱动 Bn 召回准确性自学习）+ L2（时序/因果）+ L3（行动后果，阶段②核心）。先做 L1，L2/L3 可分阶段。
- 验收：纯哈希嵌入+认知闭环下，反复共现的对象在嵌入空间形成吸引子（概念涌现）；L1 让认知压随经验累积趋向最小预测误差。
- 替换：当前无（全新增）。

### 同步要做的配套（与三块并行或紧随）
- 记忆本体重构（两张原表，第三节）——bug 6 治本。
- 驱动力管理器重做（行动评估→C*→奖惩虚能量→影响因子→叠加/衰减/冲突降差值/坚决程度）——替换 argmax(写死常数)。
- Q表接回真实路径（dialogue 路径读 Q 值参与决策）+ eligibility 升级为真 trace。
- 向量底座接对话（Zvec 接入，替换 SQLite 共现键值表近似）。
- runtime_pressure 废除，提交/停机由认知压/把握/期待驱动。
- 六阶段协议在开放对话 runtime 里真正运行（Stage 1-2 echo/successor 真跑，Stage 5 teacher-off 验收）。

### 前端 bug 修复的优先级定位
- bug 1/6 是**底层重做**的投影，根治在第 1-3 块 + 记忆本体重构。
- bug 2/5 可在底层重做的同时，用理论修法方向（焦点重采样累积 / 力导向布局）做前端改进，但完整拟人效果仍依赖底层能量驱动。
- bug 4 是纯 CSS，随时可修。
