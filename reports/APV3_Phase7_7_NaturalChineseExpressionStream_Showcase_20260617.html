APV3 Phase7.7：从“内在状态”学会自然中文表达

这一阶段把前面已经跑通的结构性内省 feeling，推进到真实中文短句 token 流。 系统先经历一种内部结构态，再观察到周围人或教师在同一时刻说出的中文表达； 经过多次共现后，teacher-off 时只给内部结构态，它能自己召回对应短句。

验收通过：5 类结构态全部召回对应中文短句

这一步为什么重要

APV3 的目标不是预先写一张“遇到问题就回答什么”的表，而是让系统像孩子一样， 在自己的内部状态和外界表达之间建立后天关联。Phase7.7 证明了这条链路已经能接上自然中文： “我还不确定”“我先记着”“我想请教一下”这类短句，不是内置规则，而是被当作普通 token 流学习到的表达材料。

1. 内部结构态出现
例如草稿不能决、工作记忆未完成、主动求教压力、惩罚痕迹、顺畅奖励。
2. 同时观察中文表达
教师或环境给出真实短句 token 流，例如“我还不确定”。
3. teacher-off 自主召回
之后不再提供短句，只由结构态触发学到的表达序列。

验证强度

5/5 五类结构性 feeling 都召回了各自中文短句。
7x+ 最弱一组目标分数也明显高于最高干扰项。
0 teacher-off replay 中没有外部表达 token 输入。

干扰项也一起进入学习环境

测试中不只喂目标短句，还同时放入低权重干扰短句，以及同一目标短句的倒序版本。 最终胜出的是目标 phrase id 与它保存下来的正确顺序，说明系统学到的不是“某几个字出现过”， 而是“这个内部结构态更常和这个表达短句整体一起出现”。

保存恢复后仍然一致

共现表与中文短句记忆都写入 runtime state，经 SQLite warm-load 后重新召回， 输出的 phrase id 与中文 token 序列保持一致。

它和常见误解的区别

这个结果的关键价值在于：中文短句不是学生侧 policy，也不是“看到不确定三个字就回答某句话”。 学生侧只拥有结构态、共现关联、短句支持度和 teacher-off 召回链路。 中文内容本身没有特殊地位，换成听觉 token、视觉符号、手势 token，同一机制也可以学习。

  • runtime 红线扫描没有 Phase7.7 中文短句。
  • runtime 没有答案表、关键词路由、旧 must_reply 通道或 student-side LLM。
  • 表达是后天观察得到的 AP-native evidence，教师可以加速，但学生侧不依赖教师策略在线作答。
